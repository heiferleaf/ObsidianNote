Spring 设计思想与核心模块笔记

一、常见设计思想

类的扩展

借助 JDK 反射机制，通过动态代理（JDK Proxy、CGLib）实现代理类的创建。常见于 AOP、MyBatis Mapper 接口实现类 Bean 的创建以及事务管理（本质也是 AOP）。

适配器模式：WebMVC 处理器适配器将处理器映射器得到的处理器所需参数进行适配，使处理器专注于业务逻辑，不必关心参数获取的具体方式。

类的创建

容器以单例模式管理 Bean（也可配置为原型）。复杂 Bean 的创建借助建造者模式，并通过工厂模式封装创建过程。

类对象之间的协作

事件订阅机制本质是观察者模式。责任链模式用于 HandlerMapping 的执行传递，以及多个 AOP 通知对同一方法的串联执行。

二、IOC 容器体系

BeanFactory 核心接口

最基础的接口是 BeanFactory，提供 Bean 的获取能力。其下有四个核心子接口：

HierarchicalBeanFactory
父子容器管理。WebMVC 中业务 Bean 在父容器，DispatcherServlet 相关 Bean（映射器、适配器、Controller 等）在子容器，Controller 可注入父容器的 Bean。
ListableBeanFactory
支持按类型获取一批 Bean，提供迭代访问能力。
ConfigurableBeanFactory
赋予容器配置能力：设置 ClassLoader、类型转换器，支持通过 Bean 后处理器和 BeanFactory 后处理器动态修改容器中的 Bean。
AutowireCapableBeanFactory
定义 Bean 的实例化与初始化方式，支持 @Autowired 属性注入。

DefaultListableBeanFactory 同时实现 ConfigurableListableBeanFactory 和 BeanDefinitionRegistry，是两套体系的核心汇聚点。

ApplicationContext 的扩展能力

仅有 BeanFactory 还不够，ApplicationContext 在此基础上补充了四项能力：

  • 资源读取：支持从 classpath、文件系统、URL 等路径加载外部配置
  • 配置环境管理：区分开发配置与部署配置，按 Profile 激活
  • 国际化支持：根据用户语言环境自动转换文本
  • 事件发布订阅：Bean 之间通过事件机制解耦协作

常用实现类：ClassPathXmlApplicationContext（XML 配置）、AnnotationConfigApplicationContext（注解配置）。

三级缓存与循环依赖

三级缓存结构

① singletonObjects（一级）— 完整的 Bean
② earlySingletonObjects（二级）— 已实例化但未完成初始化、且被其他 Bean 引用的对象
③ singletonFactories（三级）— 存放 Lambda 表达式，而非对象本身。当尝试从三级缓存获取 Bean 时，Lambda 判断该 Bean 是否需要代理：若需要，则提前执行 postProcessAfterInitialization 返回代理对象，并在 earlyProxyReferences 中标记"已提前代理"，防止后续重复代理。

关于三级缓存的几个关键细节：

  • 代理类内部持有被代理对象 target 的引用，提前代理时 target 可能尚未完成初始化，但属性注入等后续操作都作用在 target 上，代理类无需关心
  • Bean 提前代理后，正常初始化流程会跳过代理步骤，最终一级缓存中存放的是代理类
  • 构造函数引发的循环依赖无法通过三级缓存解决，因为连 Bean 实例都尚未创建
  • 原型（Prototype）Scope 的 Bean 不会放入三级缓存，因此也无法处理循环依赖

三、AOP

核心概念

AOP（面向切面编程）在 OOP 基础上，对类、属性、方法进行横向组合，在保持代码解耦的同时实现功能增强。

  • Target：被增强的对象
  • Advice：增强/通知逻辑
  • JoinPoint：连接点，包含代理对象、被代理对象、方法及参数的完整上下文
  • PointCut：切入点表达式，在 JoinPoint 集合中筛选需要增强的执行点

底层代理机制

Spring 默认使用 JDK 动态代理（基于接口），也可指定 CGLib（基于继承）。两者的主要差异：

CGLib 在代理类生成阶段略慢（需生成 Class 方法表），但方法调用时更快——通过方法索引直接定位，无需反射。

四、事务管理

Spring 事务本质是 AOP，提供三种配置方式：

  • XML 配置：注册事务管理器 Bean，通过 <tx:advice> 和 <aop:config> 定义切面与属性
  • 半注解配置：注册事务管理器 + <tx:annotation-driven> + @Transactional
  • 纯注解配置：注册事务管理器 + @EnableTransactionManagement + @Transactional

五、WebMVC

核心处理流程

DispatcherServlet 在初始化时优先使用容器中注册的 Bean，不存在则回退至默认配置组件。核心组件包括：

  • 处理器映射器（HandlerMapping）：根据请求 URL 找到对应处理器
  • 处理器适配器（HandlerAdapter）：适配方法参数，执行处理器
  • 视图解析器（ViewResolver）：解析逻辑视图名为具体视图

注解配置启动方式

Spring Web 在启动时扫描 META-INF/services/ 中指定的 ServletContainerInitializer 实现类，通过 @HandlesTypes 收集 WebApplicationInitializer 的子类并调用 onStartup。继承 AbstractAnnotationConfigDispatcherServletInitializer 即可在重写方法中指定父子容器的配置类，实现全注解开发。

常用注解

@Component@Repository@Service@Controller @Scope@Lazy@PreDestroy@PostConstruct @Autowired@Value@Resource@Qualifier @Configuration@ComponentScan@PropertySource@Primary@Profile

拦截器配置

通过 <mvc:interceptors>（XML）或实现 WebMvcConfigurer 中的 addInterceptors 方法（注解）注册拦截器，配置匹配路径，用于通用逻辑的解耦处理。

六、数据访问（以 MyBatis 集成为例）

Spring 集成 MyBatis 的核心是将 SqlSessionFactoryBean 和 MapperScannerConfigurer 注册为 Bean。所有第三方命名空间（如 mybatis:）的原理一致：容器启动时扫描自定义标签，从 META-INF/spring.handlers 中找到处理器，在处理器内部注册所需 Bean，省去手写静态工厂的步骤。
