## 🧩 三、简答题

1. 什么是函数式编程？  
	一种编程范式，核心思想是：用函数描述"做什么"而不是"怎么做"，强调函数是一等公民、避免副作用、使用不可变数据。
2. 什么是纯函数？举例说明。  
	满足两个条件：① 相同输入永远返回相同输出；② 没有任何副作用。
3. 什么是副作用？为什么要避免？  
	函数在"返回值"之外对外部世界产生的影响，比如修改全局变量、写文件、打印日志、修改入参对象等。
	避免的原因：有副作用的函数难以测试、难以并发执行、行为不可预测，代码可读性和可维护性差。
4. map 和 filter 的区别？  

| map | filter |
|---|---|---|
|作用|转换每个元素|筛选元素|
|元素数量|不变|可能减少|
|返回|新类型的 Stream|同类型的 Stream|

5. 什么是高阶函数？  
	接受函数作为参数，或者返回函数的函数。`map`、`filter`、`reduce` 都是高阶函数。
6. 为什么 Lambda 是函数对象？  
	Java 是纯面向对象语言，不存在"独立的函数"，所有东西必须是对象。Lambda 本质上是**函数式接口的匿名实现对象**，编译器会把它变成一个实现了对应接口的对象，所以它既是函数（有行为），也是对象（可以传递、存储）。

## 💻 四、代码理解题

### 1.
```java
List<Integer> nums = Arrays.asList(1, 2, 3, 4);

List<Integer> result = nums.stream()
    .filter(n -> n % 2 == 0)
    .map(n -> n * n)
    .collect(Collectors.toList());
```

问题：
1. filter 作用？  筛选出偶数
2. map 作用？  将每个元素平方
3. result 是什么？  [4, 16]

```java
Function<String, Integer> f = String::length;
```

2. 返回类型？  Integer
3. 语义？          得到输入字符串的长度

### 3.
```java
int total = 0;
List<Integer> list = Arrays.asList(1, 2, 3);

list.forEach(n -> total += n);
```

问题：
1. 是否有副作用？  有，并且这段代码编译不通过
2. 副作用在哪？      修改了外部变量total的值，同时违反了 lambda 代码体中捕获的变量必须是 effectively final 的规则
3. 如何改写为函数式？  

```java
int total = list.stream().reduce(0, Integer::sum);
```
