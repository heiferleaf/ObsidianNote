## 什么是操作系统

操作系统是介于底层硬件和应用软件之间的系统软件，负责管理底层硬件资源，控制程序运行，为应用程序和用户提供服务。是计算机中最基本的系统软件。

> [!NOTE] 如果没有操作系统，PC机还能做什么
> 
> CPU通电后还是可以运行，但是会有以下变化：
> 1. 面向机器语言编码：没有操作系统提供的编译器和库，没有办法编写高级语言，需要写二进制的机器语言，来让CPU运行
> 2. 直接使用底层硬件资源：操作系统可以管理底层硬件资源，没有OS，应用程序需要直接面向复杂的硬件编码
> 3. 单进程运行：OS可以管理应用程序运行，没有OS，无法实现多进程的调度，内存中只能存在一个进程的资源
> 4. 硬件资源脆弱：OS通过将CPU的运行状态分为用户态和内核态，控制应用程序对于硬件资源的访问。没有OS，应用程序对于硬件的不正确使用，可能会损坏硬件


## 操作系统的特性

1. 并发：同一时间间隔有多个程序运行，具有调度处理的能力。
	> 并行：同一时间时刻有多个程序运行，需要多个处理器
2. 虚拟：将物理上的资源分为若干逻辑上的独立物。
	> 1. CPU：通过OS的调度器，让每一个应用进程认为自己完全占有CPU
	> 2. 内存资源：通过MMU和页表，让每一个应用进程认为自己有独立的内存空间
	> 3. 硬件设备：通过统一的接口来调用设备
3. 共享：应用程序共享硬件资源。
4. 异步：由并发决定的，进程执行过程的时间不确定性（何时暂停、何时推进、何时完成）

## 操作系统的发展


| 发展历程   | 特点                               | 优缺点                      |
| ------ | -------------------------------- | ------------------------ |
| 人工处理   | 程序员手动操控硬件                        | 麻烦，CPU的等待时间大部分取决于人的操作快不快 |
| 脱机处理   | 外围机控制输入输出                        | 减少了CPU的等待时间              |
| 单道批处理  | 单道、顺序、自动                         | CPU利用率较低（对于长IO任务）        |
| 多道批处理  | 多道、无序、并发执行                       | 资源利用率和吞吐量高，但是没有交互能力      |
| 分时操作系统 | 多路（多个用户终端和一个主机连接）、分时（主机分配时间片给终端） | 区分不了任务的优先级               |
| 实时操作系统 | 多路、即时（保证一个任务可以在指定时间内完成或者做出响应）    | 分为硬实时和软实时                |



## 操作系统的功能

1. 管理硬件资源
	硬件资源指CPU、内存、IO设备、硬盘。
	因为资源是有限的，所以就要进行管理，管理可以分为资源的分配和资源的回收。
	- 资源分配：以CPU为例，当多个进程请求CPU，如何决定分配给哪一个进程；以内存为例，当内存资源不足，如何决定替换那一块内存物理页
	- 资源回收：以内存为例，连续分配策略下，如何回收内存，让内存碎片尽可能少
2. 为应用程序提供服务
	为应用提供系统调用，来访问硬件资源。
	如果没有这个功能，应用就需要直接面向硬件编程，复杂且容易损坏硬件。
3. 控制程序运行
	通过调度器来调度程序的运行

> 综合来看，可以概括为：
> 1. 进程调度能力
> 2. 内存管理能力：负责内存的分配和回收
> 3. 硬件通信能力：管理硬件，为进程和硬件之间提供通信
> 4. 系统调用能力

## 操作系统运行环境

### CPU 的状态

操作系统本质上是一个系统软件，可以管理底层资源、控制程序运行，所能执行的操作更多、权限更高。而对于用户程序，需要借助OS，才能完成对底层资源的访问，拥有的权限比较低，这也是为了保护OS的安全。

所以CPU的运行的状态分为：
- 用户态：只能访问用户程序自己的内存空间，不能执行特权指令、改变机器状态、使用所有资源
- 内核态：可以访问所有的内存空间、使用硬件资源、特权指令、改变机器状态

从用户态进入内核态的方式：**中断、异常**

### 中断、异常

| 类型  | 触发方式                             |
| --- | -------------------------------- |
| 中断  | 外部硬件设备触发、和CPU正在运行的程序无关，一般是时钟、键盘等 |
| 异常  | 来源于执行的指令                         |

异常可以进一步分类
- 故障：执行指令出错，除零、溢出、缺页
- 陷入：指令发出的系统调用
- 终止：硬件出现无法继续执行的故障，比如存储器校验出错、控制器出错

中断触发的流程：
1. 中断响应：硬件发出中断请求，CPU发现后，停止运行当前程序
2. 现场保护：==关闭中断使能==，将当前CPU运行进程的上下文保存到PCB、进程栈帧中
3. 中断处理：根据中断向量，CPU 运行对应的处理程序
4. 现场恢复：从保存的上下文恢复到对应的寄存器，==打开中断使能位==

陷入的处理流程：
1. CPU 运行的指令触发系统调用，将系统调用所需的参数，通过寄存器、堆栈进行传递
2. 用户态进入内核态，现场保护
3. 根据中断向量号判断出是系统调用
4. 根据传入的参数，判断是什么系统调用
5. 根据传入的参数，执行系统调用
6. 现场恢复，返回用户态代码

> 有的库函数，就是对系统调用的封装
> 中断后，执行的中断的下一条指令；异常后，如果是缺页异常，执行缺页的指令
