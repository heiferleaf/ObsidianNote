> **“根基的深度，决定了大厦的高度。” —— 打好基础，是精通任何技能的起点。**

### **Java 基础核心概念梳理**

#### **1. JDK 与 JRE 关系**

*   **JDK (Java Development Kit)**
    *   **定位**：Java **开发**工具集。
    *   **包含关系**：**JDK = JRE + 开发工具**（如 `javac` 编译器、`java` 解释器）。
    *   **场景**：需**编写和运行**Java程序时安装。

*   **JRE (Java Runtime Environment)**
    *   **定位**：Java **运行时**环境。
    *   **包含关系**：**JRE = JVM + Java核心类库**。
    *   **场景**：仅需**运行**Java程序时安装。

*   **实践提示**：现代JDK版本已内置JRE，区分概念在设置环境时仍有意义。

#### **2. Java 语言特性**

*   **工作流程**：
    1.  **编译**：`javac` 将 `.java` 源码编译成平台无关的 **`.class` 字节码**。
    2.  **运行**：`java` 命令启动**JVM**，将字节码解释/编译为本地机器码执行。

*   **核心价值**：**“一次编译，到处运行”**，前提是目标系统有匹配的JVM。

*   **实践提示**：JVM使用JIT编译器优化热点代码，保障了运行效率。

#### **3. 数据类型与运算注意事项**

*   **整型强制转换**
    *   **规则**：大范围类型值赋给小范围变量需显式转换：`目标类型 变量 = (目标类型) 值;`。
    *   **溢出风险**：转换后值超出目标类型范围时，会发生**溢出**（循环取值），而非截断为最大值。本质还是二进制的截断，解读为有符号数或者无符号数
        *   *示例*：`(byte) 300` 结果为 `44`。

*   **浮点数 (`float` & `double`)**
    *   **字面量**：默认 `3.14` 为 `double` 类型；声明 `float` 必须加 `f` 后缀：`float f = 3.14f;`。
    *   **运算限制**：**不能**进行移位运算 (`<<`, `>>`, `>>>`)。

*   **浮点转整型规则**
    *   **规则**：`(int)` 强制转换采用 **“向零取整”**，直接舍弃小数部分。
        *   *示例*：`(int) 3.7 -> 3`； `(int) -3.7 -> -3`。
    *   **对比**：与 `Math.floor()`（向下取整）和 `Math.ceil()`（向上取整）不同。

*   **实践提示**：警惕强制转换的溢出与精度丢失；需四舍五入时使用 `Math.round()`。
