## 数据结构特征

1. 以树状形式组织堆
2. 根节点比子节点都大（大根堆） | 根节点比子节点都要小（小根堆）

## 关键算法（以大根堆为例）

1. 下沉操作 `shiftDown(int i)`
	```Java
	void shiftDown(int i) {
		int num = i对应结点的值
		while(i不是叶子结点) {
			int j = i 的左节点索引
			if(j不是最后一个结点索引 && i的右节点 > 左节点)
				j = i 的右节点
			if(i结点值 > j节点值) {
				break;
			}
			i结点的值 = j结点的值
			i = j
		}
		i结点值 = num
	}
	```
	> 关键在于提前停止循环。
	> - 前提：对结点i进行下沉操作之前，i结点处的子树已经是堆
	> - 前提决定了当i找到合适位置的时候，整体是堆
	
2. 上浮操作 `shiftUp(int i)`
```Java
void shiftUp(int i) {
	while(i不是根节点) {
		if(i结点值 < i的父节点值) break;
		交换 i 和他的 父节点
		i = 父节点索引
	}
}
```


## 堆的建立

建立堆，可以考虑
- 从最后一个非叶子结点开始，执行下沉操作
- 从第一个根节点开始，到最后一个叶子节点，执行上浮操作
第一种时间复杂度更加低，因为大量的数据执行算法时间少，少量数据执行时间多。

可以从数学角度证明
第一种的时间复杂度是 $O(n)$, 第二种时间复杂度是 $O(n\log(n))$
