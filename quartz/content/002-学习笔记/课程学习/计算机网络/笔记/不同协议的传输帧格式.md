## 1. 物理层/数据链路层：MAC帧

### 以太网帧（Ethernet II）格式

| 字段            | 长度        | 描述                                     |
| ------------- | --------- | -------------------------------------- |
| 前导码(Preamble) | 7字节       | 10101010序列，用于物理层同步，因为发送方和接受方需要对时钟的信息同步 |
| 帧开始符(SFD)     | 1字节       | 10101011，标记帧开始                         |
| 目的MAC地址       | 6字节       | 接收方MAC地址                               |
| 源MAC地址        | 6字节       | 发送方MAC地址                               |
| 类型/长度         | 2字节       | 指示上层协议类型（如IP=0x0800，ARP=0x0806）        |
| 数据/Payload    | 46~1500字节 | 上层协议数据，最小46字节                          |
| 填充(Padding)   | 变长        | 若数据不足46字节，需要填充                         |
| 帧校验序列(FCS)    | 4字节       | CRC32校验，用于错误检测                         |

- 整个MAC帧长度范围：64~1518字节
- MTU（最大传输单元）：1500字节（数据区最大值）
- 最小帧长：64字节（不包括前导码和帧开始符）

## 2. MAC地址解析

MAC地址是48位（6字节）的物理地址，以十六进制表示，如：`00:1A:2B:3C:4D:5E`

### MAC地址结构
- **前24位（3字节）**：厂商标识符(OUI)，由IEEE分配给设备制造商
- **后24位（3字节）**：由制造商分配的唯一序列号

### MAC地址特点
- **全球唯一**：理论上每个网络接口都有唯一MAC地址
- **烧录在硬件中**：通常由制造商预设在网卡ROM中
- **平面结构**：不分层，不具有网络拓扑信息
- **本地有效**：只在同一广播域内有效

### MAC地址特殊值
- **单播地址**：第一字节最低位为0
- **多播地址**：第一字节最低位为1（如01:00:5E:XX:XX:XX是IPv4多播）
- **广播地址**：全为1（FF:FF:FF:FF:FF:FF）

## 3. 网络层：IP地址与IP数据报

### IP地址含义
- IP地址是网络层的逻辑地址，用于全局寻址和路由
- IPv4地址为32位（4字节），表示为点分十进制（如192.168.1.1）
- 分为网络部分和主机部分，通过子网掩码区分

### IP数据报格式（IPv4）

| 字段                     | 长度(位) | 描述                          |
| ---------------------- | ----- | --------------------------- |
| 版本(Version)            | 4     | IPv4为4                      |
| 首部长度(IHL)              | 4     | 以4字节为单位，最小值为5（20字节）         |
| 服务类型(TOS)              | 8     | 区分服务质量                      |
| 总长度(Total Length)      | 16    | IP数据报总长度（头部+数据），最大65535字节   |
| 标识(Identification)     | 16    | 用于分片重组标识                    |
| 标志(Flags)              | 3     | 用于控制和标识分片（DF-不分片，MF-更多分片）   |
| 片偏移(Fragment Offset)   | 13    | 指示分片在原始数据报中的位置              |
| 生存时间(TTL)              | 8     | 路由器转发次数限制，每经过一个路由器减1        |
| 协议(Protocol)           | 8     | 上层协议号（TCP=6，UDP=17，ICMP=1等） |
| 头部校验和(Header Checksum) | 16    | 仅校验IP头部                     |
| 源IP地址                  | 32    | 发送方IP地址                     |
| 目的IP地址                 | 32    | 接收方IP地址                     |
| 选项(Options)            | 变长    | 可选字段，很少使用                   |
| 填充(Padding)            | 变长    | 确保首部长度为4字节整数倍               |
| 数据                     | 变长    | 上层协议数据                      |

### 协议号对应表（常见）

| 协议名称 | 协议号 |
|---------|--------|
| ICMP    | 1      |
| IGMP    | 2      |
| TCP     | 6      |
| UDP     | 17     |
| GRE     | 47     |
| ESP     | 50     |
| AH      | 51     |
| OSPF    | 89     |

## 4. 特殊IP地址含义

| IP地址/范围 | 含义 |
|------------|------|
| 0.0.0.0 | 表示本机、未指定地址或默认路由 |
| 127.0.0.0/8 | 环回地址范围，其中127.0.0.1表示本机 |
| 10.0.0.0/8 | A类私有地址 |
| 172.16.0.0/12 | B类私有地址 |
| 192.168.0.0/16 | C类私有地址 |
| 169.254.0.0/16 | APIPA地址（无DHCP时自动配置） |
| 224.0.0.0/4 | D类地址（多播地址范围） |
| 255.255.255.255 | 有限广播地址（本地网络） |
| 网络号.全1 | 定向广播地址（如192.168.1.255） |
| 网络号.0 | 网络地址，表示整个网段 |

## 5. ICMP协议

ICMP协议位于IP层之上，用于网络诊断和错误报告。ICMP报文封装在IP数据报中，协议号为1。

### ICMP报文格式

| 字段            | 长度(字节) | 描述           |
| ------------- | ------ | ------------ |
| 类型(Type)      | 1      | 定义ICMP报文类型   |
| 代码(Code)      | 1      | 进一步细分类型      |
| 校验和(Checksum) | 2      | 校验整个ICMP报文   |
| 内容            | 变长     | 根据类型和代码不同而变化 |

### 常见ICMP类型

| 类型  | 代码   | 描述                              |
| --- | ---- | ------------------------------- |
| 0   | 0    | 回显应答（Echo Reply，Ping响应）         |
| 3   | 0-15 | 目的不可达（如0=网络不可达，1=主机不可达，3=端口不可达） |
| 5   | 0-3  | 重定向                             |
| 8   | 0    | 回显请求（Echo Request，Ping请求）       |
| 11  | 0-1  | 超时（0=TTL超时，1=分片重组超时）            |

## 6. 路由协议：RIP

RIP是基于距离矢量的路由协议，使用UDP作为传输层协议，端口号520。

### RIP在数据包中的位置
```
[以太网帧头] → [IP头(协议=17)] → [UDP头(源端口=520,目标端口=520)] → [RIP数据]
```

### RIP数据格式

| 字段              | 长度(字节) | 描述                |
| --------------- | ------ | ----------------- |
| 命令(Command)     | 1      | 1=请求，2=响应         |
| 版本(Version)     | 1      | 1或2，表示RIP版本       |
| 保留字段            | 2      | 置为0               |
| 地址族标识(AFI)      | 2      | 通常为2（IP地址）        |
| 路由标记(Route Tag) | 2      | RIPv2使用，区分内部/外部路由 |
| IP地址            | 4      | 目的网络IP            |
| 子网掩码            | 4      | RIPv2使用，RIPv1不包含  |
| 下一跳             | 4      | RIPv2使用，RIPv1不包含  |
| 度量值(Metric)     | 4      | 跳数，1-15有效，16表示不可达 |

- 每个RIP消息最多包含25个路由条目
- 每个条目20字节（RIPv2）
- 最大更新间隔：30秒

## 7. 路由协议：OSPF

OSPF是链路状态路由协议，直接封装在IP数据报中，协议号为89。

### OSPF在数据包中的位置
```
[以太网帧头] → [IP头(协议=89)] → [OSPF数据]
```

### OSPF基本报文头格式

| 字段                   | 长度(字节) | 描述                                         |
| -------------------- | ------ | ------------------------------------------ |
| 版本(Version)          | 1      | OSPF版本号，通常为2（OSPFv2）                       |
| 类型(Type)             | 1      | 1=Hello，2=数据库描述，3=链路状态请求，4=链路状态更新，5=链路状态确认 |
| 报文长度(Length)         | 2      | 包括OSPF头的总长度                                |
| 源路由器ID               | 4      | 发送方路由器ID                                   |
| 区域ID(Area ID)        | 4      | 报文所属的区域                                    |
| 校验和(Checksum)        | 2      | 整个OSPF报文的校验和                               |
| 认证类型(Auth Type)      | 2      | 0=无认证，1=简单认证，2=MD5认证                       |
| 认证数据(Authentication) | 8      | 根据认证类型变化                                   |

### OSPF Hello包格式（在基本报文头之后）

| 字段 | 长度(字节) | 描述 |
|------|-----------|------|
| 网络掩码(Network Mask) | 4 | 接口所在网段的子网掩码 |
| Hello间隔(Hello Interval) | 2 | 发送Hello包的时间间隔（秒） |
| 选项(Options) | 1 | 特殊功能标志位 |
| 路由器优先级(Rtr Pri) | 1 | 用于DR/BDR选举 |
| 路由器失效时间(Dead Interval) | 4 | 邻居失效时间（秒） |
| 指定路由器(DR) | 4 | 发送方所在网段的DR路由器ID |
| 备用指定路由器(BDR) | 4 | 发送方所在网段的BDR路由器ID |
| 邻居(Neighbor) | 变长 | 已知邻居路由器ID列表 |

## 8. 路由协议：BGP-4

BGP是自治系统间路由协议，使用TCP作为传输层协议，端口号179。

### BGP在数据包中的位置
```
[以太网帧头] → [IP头(协议=6)] → [TCP头(源端口=可变,目标端口=179)] → [BGP数据]
```

### BGP消息公共头格式

| 字段 | 长度(字节) | 描述 |
|------|-----------|------|
| 标记(Marker) | 16 | 全为1，用于认证 |
| 长度(Length) | 2 | BGP消息总长度（19-4096字节） |
| 类型(Type) | 1 | 1=OPEN，2=UPDATE，3=NOTIFICATION，4=KEEPALIVE，5=ROUTE-REFRESH |

### BGP OPEN消息格式

| 字段 | 长度(字节) | 描述 |
|------|-----------|------|
| 版本(Version) | 1 | BGP版本号，通常为4 |
| 自治系统号(AS) | 2或4 | 发送方的AS编号 |
| 保持时间(Hold Time) | 2 | 保持连接的最大时间间隔（秒） |
| BGP标识符(BGP Identifier) | 4 | 发送方的路由器ID |
| 可选参数长度 | 1 | 可选参数的总长度 |
| 可选参数 | 变长 | TLV格式的可选参数 |

### BGP UPDATE消息格式

| 字段 | 长度(字节) | 描述 |
|------|-----------|------|
| 不可达路由长度 | 2 | 撤销路由信息的长度 |
| 撤销路由 | 变长 | 不再可达的路由前缀 |
| 路径属性总长度 | 2 | 路径属性的总长度 |
| 路径属性 | 变长 | 如Origin、AS_PATH、NEXT_HOP等 |
| 网络层可达性信息(NLRI) | 变长 | 新通告的可达目的网络 |

---

通过这份笔记，可以清晰了解从物理层到应用层各种协议的帧格式和数据位含义，这对于网络故障排查、协议分析和理解计算机网络通信原理都非常有帮助。各协议之间层次分明，相互依赖，共同构建了现代计算机网络通信的基础。
