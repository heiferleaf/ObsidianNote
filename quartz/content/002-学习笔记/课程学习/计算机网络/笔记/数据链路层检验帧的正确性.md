2. 对于帧丢失
	发送数据帧后，启动一个超时计时器，没有在指定时间内收到接受方的确认帧，就进行重传
3. 对于帧重复
	使每一个数据帧带上不同的发送序号。每发送一个新的数据帧就把它的发送序号加1。
	若接收方收到发送序号相同的数据帧，就表明出现了重复帧，这时应丢弃重复帧。
	同时接受方也要像发送方发送确认帧

综上，发送方发送一帧，启动计时器，等待应答：
- 肯定应答：发新帧
- 否定应答或者无应答：重传

- ACK1表示“0号帧已收到，现在期望接收1号帧”；
- ACKO表示“1号帧已收到，现在期望接收0号帧”

# 停止-等待协议实现算法（链路层的点对点协议）

## 核心概念

- **V(S)**：发送状态变量，表示下一个要发送的数据帧编号（0或1）
- **V(R)**：接收状态变量，表示期望接收的数据帧编号（0或1）
- **N(S)**：当前发送帧的序号
- **N(R)**：当前接收帧的序号

---

## 发送方算法流程

### 初始化阶段

1. **取出数据**：从主机获取一个数据帧
2. **设置序号**：V(S) ← 0（第一个数据帧编号为"0"）
3. **构造数据帧**：N(S) ← V(S)，将数据帧送到发送缓存
4. **初始化重传**：设置重传次数Count

### 发送阶段

5. **发送数据帧**：
    - 如果Count > 0：发送缓存中的数据帧，Count减1
    - 如果Count = 0：报告错误，停止传输
6. **启动定时器**：设置超时计时器tout

### 等待应答阶段

7. **等待三种事件**之一：
    - 收到肯定应答帧ACK
    - 收到否定应答帧NAK
    - 超时计时器时间到

### 处理应答

8. **收到ACK**：
    
    - 从主机取新的数据帧
    - 更新发送序号：V(S) ← [1-V(S)]（0变1，1变0）
    - 转到步骤3继续发送下一帧
9. **收到NAK**：转到步骤4（重传当前数据帧）
    
10. **超时**：转到步骤4（重传当前数据帧）
    

---

## 接收方算法流程

### 初始化阶段

1. **设置期望序号**：V(R) ← 0（期望接收编号为0的数据帧）

### 接收处理阶段

2. **等待帧到达**
    
3. **收到数据帧时检查**：
    
    - **检查位错**：如果有位错 → 转到步骤8（发送NAK）
    - **检查序号**：如果N(R) ≠ V(R) → 转到步骤7（丢弃重复帧）

### 正确接收处理

4. **序号匹配**：N(R) = V(R)，说明是期望的帧
5. **提取数据**：将数据帧中的数据送交主机
6. **更新期望序号**：V(R) ← [1-V(R)]（准备接收下一数据帧）
7. **发送ACK**：发送肯定应答帧，转到步骤2

### 异常处理

8. **发送NAK**：发送否定应答帧，转到步骤2

---

## 关键理解要点

### ACK的含义

- **ACK中包含的序号 = 当前V(R)的值**
- **表示期望接收的下一帧编号**
- 例如：发送ACK1表示"期望接收1号帧"（说明0号帧已正确接收）

### 序号更新机制

- **发送方**：收到正确ACK后，V(S) ← [1-V(S)]
- **接收方**：正确接收帧后，V(R) ← [1-V(R)]
- **实现0和1之间的循环**：0 → 1 → 0 → 1...

### 错误处理策略

1. **位错误**：接收方发送NAK，发送方重传
2. **丢失/超时**：发送方重传当前帧
3. **重复帧**：接收方检查序号，丢弃重复帧但重发上次ACK

### 可靠性保证

- **确认重传机制**：每帧必须收到ACK才发送下一帧
- **超时重传**：防止帧丢失或ACK丢失
- **序号检查**：防止重复帧被重复处理


# 停止-等待协议性能分析

## 基本时间参数

### 关键时间定义

- **TD**：数据帧发送时间 = L / R （L为帧长，R为信道带宽）
- **TP**：单向传播时延
- **TA**：ACK帧发送时间（通常很小，可忽略，有时候ACK比较大，也不能忽略，还是要看题）
- **总往返时间RTT** = 2TP + TA ≈ 2TP

---

## 信道利用率计算

### 理想情况（无差错）

**信道利用率 U = 有效传输时间 / 总循环时间**

```
U = TD / (TD + RTT) = TD / (TD + 2TP)
```

### 用带宽时延积表示

设**带宽时延积 a = TP × R / L**，则：

```
U = 1 / (1 + 2a)
```

### 性能分析

- **当 a << 1 时**：U ≈ 1 （信道利用率高）
- **当 a >> 1 时**：U ≈ 1/(2a) （信道利用率很低）

**物理含义**：

- a 表示传输一帧期间，信道上能容纳多少帧
- a 越大，说明传播时延相对越长，利用率越低

---

## 考虑差错的信道利用率

### 参数定义

- **P**：帧出错概率
- **成功传输概率** = 1 - P

### 平均重传次数

由于几何分布，平均传输次数为：

```
E[重传次数] = 1 / (1 - P)
```

### 考虑差错的利用率

```
U = (1 - P) × TD / (TD + 2TP) = (1 - P) / (1 + 2a)
```

---

## 吞吐量计算

### 定义

**吞吐量 S = 单位时间内成功传输的数据量**

### 基本公式

```
S = U × R × (1 - P)
```

其中：

- U：理想信道利用率 = 1/(1 + 2a)
- R：信道带宽
- (1 - P)：成功传输概率

### 完整表达式

```
S = R × (1 - P) / (1 + 2a)
```

---

## 数值计算示例

### 例题1：基本计算

**条件**：

- 帧长 L = 1000 bit
- 带宽 R = 1 Mbps = 10⁶ bps
- 传播时延 TP = 20 ms
- 出错概率 P = 10⁻⁴

**计算过程**：

1. **数据帧发送时间**：TD = L/R = 1000/(10⁶) = 1 ms
2. **往返时延**：RTT = 2TP = 40 ms
3. **带宽时延积**：a = TP × R / L = 20×10⁻³ × 10⁶ / 1000 = 20
4. **理想利用率**：U = 1/(1 + 2×20) = 1/41 ≈ 2.44%
5. **实际利用率**：U实际 = (1 - 10⁻⁴)/41 ≈ 2.44%
6. **吞吐量**：S = 10⁶ × (1 - 10⁻⁴)/41 ≈ 24,390 bps

---

## 关键结论

### 性能特点

1. **停止-等待协议效率低**，特别是在高带宽时延积环境中
2. **传播时延是主要瓶颈**，而不是传输时延
3. **适用于短距离、低速率通信**

### 改进方向

1. **流水线传输**：连续发送多个帧
2. **差错控制优化**：减少不必要的重传
3. **自适应参数调整**：根据网络状况动态调整

### 实际应用

- **局域网环境**：性能可接受
- **广域网/卫星通信**：性能很差，需要改进协议
- **物联网低功耗场景**：简单可靠，仍有应用价值

# 滑动窗口（TCP协议）

- 选择重传
	不需要按序传送，帧用n位编号，窗口大小最大 $W_R\le2^n/2$
	发送方需要统计收到的ACK，收到N个后推进N，收到NAK时，重传对应的帧
	接受方对收到的每个帧帧进行检错和排序，正确发送ACK，错误发送NAK
- 全部重传
	需要按序传送，帧用n位编号，窗口大小最大 $W_R \le 2^n -  1$
	检测到第i帧出错，就从第i帧开始重传

# 接收端接受信息的同步


同步方式是指通信双方在数据传输过程中，对信息边界和时序的共同理解与协调。由于没有统一时钟，实际工作中常为异步同步。同步机制是数据链路层可靠传输的基础。

## 一、同步的分类
同步包括以下几种类型：
1. **位同步**  
2. **字符同步**
3. **帧同步**
4. **状态同步**

---

## 二、位同步

### 概念
- 保证接收方能准确区分和恢复每一个比特位。
- 是最基础的同步要求。

### 方法
1. **插入导频法**  
   - 在发送信号中定期插入专门的同步信号（导频位），接收方据此实现位同步。
2. **自同步法**  
   - 采用特定的编码方法（如曼彻斯特编码、nB/mB编码），使信号本身带有时钟信息，方便接收端同步。

---

## 三、字符同步

### 概念
- 以字符为单位进行同步，确保接收方能区分字符的起止。

### 方法
- 采用起始位、数据位、可选的奇偶校验位和停止位组成一个字符传输单元。
    - **起始位（1位，恒0）**：表示字符开始。
    - **数据位（7/8位）**：实际数据内容。
    - **奇偶校验位（可选，1位）**：用于错误检测。
    - **停止位（1/1.5/2位，恒1）**：表示字符结束。
- 通过每个字符的结构，接收端可同步每个字符的边界。
- **时钟漂移的影响**：由于时钟不一致，累计漂移会导致采样错误。设计起始位和停止位可缓解漂移影响。

---

## 四、帧同步

### 概念
- 数据传输时，以“帧”为单位，需保证接收方能正确识别每一帧的起止位置。

### 方法
1. **定界符法（Flag/Delimiter）**  
   - 在帧的开始和结束插入特定的比特序列（如01111110），接收方根据该序列识别帧的边界。
   - 需注意“比特填充”技术，防止数据内容与定界符混淆。
2. **长度字段法**  
   - 在帧头部加上表示帧长度的字段，接收方通过读取长度来确定帧的范围。
3. **其它方法**  
   - 还可采用不同的编码或协议实现帧同步。

---

## 五、状态同步

### 概念
- 除了数据本身外，通信双方还需同步协议的状态信息，确保数据交互的正确性。

### 作用
- 状态同步保证双方在协议的同一状态下协同操作，如发送窗口、接收窗口的管理、超时重传、流量控制等。
- 若因误码、丢包等导致状态失步，需通过重置、重同步等手段恢复同步。

### 例子
- 滑动窗口协议中，双方需要同步“已确认”、“待发送”等状态。
- 停止等待协议中，双方通过ACK/NACK保持交互状态的同步。

---

## 六、总结对比

| 同步类型 | 作用               | 实现方法       |
| ---- | ---------------- | ---------- |
| 位同步  | 区分每一比特           | 导频/自同步编码   |
| 字符同步 | 区分每个字符边界         | 起始位/停止位    |
| 帧同步  | 区分每一帧数据的起止       | 定界符/长度字段   |
| 状态同步 | 协议过程协同，保证数据交互正确性 | 协议机制/重同步手段 |
|      |                  |            |

---

**同步机制是数据链路层可靠传输的基础，只有实现了各级同步，才能实现正确的数据通信。**
