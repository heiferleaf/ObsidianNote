# 漏洞挖掘技术分类

## 静态分析方法

不需要运行程序，分析程序潜在的漏洞。**分析效率高、资源开销小**。
代表方法：
- 符号执行
- 污点传播分析
- 词法分析
- 数据流分析

## 动态分析方法

需要运行程序，发现程序的漏洞。**查找准确率高**。
代表方法：
- 动态符号执行
- 动态污点分析
- 模糊测试

# 符号执行

## 概念

**使用符号代替具体的变量，收集程序语义，探索程序可达路径，分析潜在漏洞**。
动态符号执行，在实际执行的同时，进行符号执行，在保证测试精度的前提下，提高执行效率。

## 步骤

1. 变量符号化：把外部输入的数值转为符号，变量通过符号的表达式来表示
2. 模拟程序执行
	1. 对于运算语句，把变量用符号表达式替换
	2. 对于分支语句，记录不同的路径和进入路径的约束条件
3. 约束求解：对所有的路径，通过其约束条件进行求解，**判断路径是否可达，对可达路径生成测试输入**


# 污点分析

## 概念

**通过标记程序的外部输入数据或内部数据位污点，追踪程序流程中，污点数据的处理流程，完成对程序的分析**

污点分析分为静态的污点传播分析，和动态的污点分析。静态分析通过模拟程序运行，对污点传播进行分析，动态分析通过实际运行程序，传播并追踪污点数据。

污点分析常用来挖掘注入类的漏洞，如SQL注入、SOAP注入
## 步骤

1. 标记**污点源**
2. 确定**污点传播规则**：包括污点的扩散、清除。一般的计算、赋值语句是扩散；常值赋值是清除
3. 进行**污点检测**：在程序的敏感位置，对污点进行判定。常见的敏感位置包括程序的跳转、函数调用

# 词法分析

## 概念

对代码进行基于文本或者字符标识的匹配，进行分析，查找符合特定特点和词法规则的危险函数、api、语句组合。

实现比较简单，执行效率高，但是会有很多的漏测和误报

# 模糊测试

属于黑盒测试的一种，向程序输入大量**畸形数据**，并检测异常来发现存在的漏洞。

一般步骤：
1. **确认测试对象和测试数据**：根据目标程序，确定可能的输入数据
2. **生成模糊测试数据**：根据确定的输入数据，选用测试数据生成算法，生成畸形数据
3. **测试畸形数据**
4. **检测程序异常情况**
5. 确定可利用性：确定异常是否一个会被利用的漏洞，不是必须的操作

模糊测试因为测试数据是基于合法数据畸变生成，无法保证不同情况的覆盖率。衍生出智能模糊测试（白盒测试），在通过符号分析、污点分析等程序分析手段之后，针对不同的执行路径生成畸变测试数据。


# 程序切片

对于大规模的程序，会因为执行路径过于复杂，导致污点传播等比较难追踪。程序切片专注于将某一个变量的完整路径，从程序中切片出来，**简化分析对象**。

## 步骤

分析程序：通过 CFG（Control Flow Graph 控制流图）、PDG（Program Dependency Grpah 程序依赖图）、SDG（System Dependency Graph 系统依赖图），对程序的执行路径和数据来源路径可视化。

> PDG 的两个关键：数据流程、控制流程

根据分析得到的图，对特定的变量进行静态切片或者动态切片
- **静态切片**：考虑变量所有的可能执行路径
- **动态切片**：在特定条件下，变量的执行路径
