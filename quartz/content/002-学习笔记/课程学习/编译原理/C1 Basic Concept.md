# 课程理解

## 程序的翻译

我们平时说的代码，包括
- 源代码（一般是高级语言代码）
- 汇编代码
- 机器代码（01二进制）

计算机只认识01二进制，如何执行高级语言呢。答案：通过翻译，将高级语言翻译成机器代码。

所以机器执行高级语言的两个阶段：
1. 翻译代码
2. 运行代码

其中翻译的方式大致分为编译和解释
- 编译方式：
	- 编译：将高级语言源代码，通过编译程序，得到汇编代码或者机器代码
	- 汇编：将汇编代码，通过汇编程序，得到机器代码
	> 1. 编译程序输出的结果可能是汇编代码，有可能是机器代码
	> 2. 编译程序和高级语言种类、机器的硬件架构强关联
	> 3. 使用编译作为翻译方式，最终一定以源代码为输入，得到目标代码
	> 4. ![[QQ_1773229846282.png]]
- 解释方式
	- 按照源代码的操作序列定义，与提供的输入，逐句的解释并执行
	> 1. 使用解释作为翻译的方式，不会得到目标代码
	 2. ![[QQ_1773229858546.png]]
> ==是否生成目标代码==就是编译和解释这两种方式的本质区别
- 当前还有很多语言是编译 + 解释类型的，比如Java，他们的一个特点，就是编译输出的目标程序是字节码

## 编译程序的工作过程

### （一）词法分析

目前的理解是对于文本字符的切割方式，具体点就是我们认为怎么样的文本可以作为一个词，**这个怎么样认为**就是一种规则认识，也就是法，词法。

词法分析的目标，是将源代码程序，处理为一序列的 token（词法单元），提取出 token 的属性特征（类型是数、标识符、赋值符号、运算符号），得到一张符号表
![[QQ_1773230540723.png]]
### （二）语法分析

就是根据词法分出来的词，怎么样是合法的。这个怎么样，就是语法定义的。

> 在以前的数据库内核实现中，语法分析器，就是把sql语句中的token，按照sql的类型，分为不同的语法树，语法树的结构，就是语法的体现。

语法的描述，通常有几种方式
1. 自然语言：比如 ==标识符== 是由一个==字母后续带有若干数字或字母==组成的符号串
2. 语法图（Syntax Graph）
	![[QQ_1773553105985.png]]
	> 左侧说明标识符的语法结构，右侧说明字母和数据的语法结构
3. ==BNF范式== ： <标识符> ::= <字母>|<标识符><字母>|<标识符><数字>
	拓展的 BNF范式： <标识符＞ ::= <字母>{<字母>|<数字>}

语法分析得到的语法树，每一个结点都是一个运算（也代表运算之后的结果），结点的子节点，就是当前结点代表运算的分量。

![[QQ_1773230567833.png]]

### （三）语义分析

语义分析的一个重要工作是类型检查，检查标识符的类型，类型和运算是否匹配，识别运算的含义并做处理
![[QQ_1773230727041.png]]
图示的处理中，发现id3的类型是浮点数，number的值是整型，于是插入浮点数转换的子节点

### （四）中间代码生成

根据语法树，生成中间代码，为后续生成目标代码做铺垫
![[QQ_1773230902849.png]]
> 图中的四元式，对应到等式(如2. t2 = id3 * t1)
> - 等式右侧只有一个运算符

### （五）中间代码优化

这里的优化往往是机器无关的，前面说到编译程序和语言、机器架构强关联，需要注意一下。
代码优化的目标一般是：
- 更短的代码表示
- 更快的执行速度
![[QQ_1773231120691.png]]
> 1. 将浮点类型转换进行优化，通过将源代码的60直接改为60.0
> 2. 省略了无必要的中间寄存器表示

### （六）目标代码生成

将中间形式的代码，转换成目标代码，和语言、机器架构强关联
![[QQ_1773231266499.png]]

### （七）表格管理

登录源程序中出现的每一个 token 以及 token 的属性，有的token属性需要在各个阶段分别填入


| NMAE | TYPE | ... | VAL | ADDR |
| ---- | ---- | --- | --- | ---- |
|      |      |     |     |      |

### （八）出错处理

编译过程中的出错可以分别语法出错和语义出错
1. 语法出错：对于语法出错，是指源代码的词法单元不符合规则，或者 token 之间的组成不满足语法结构定义。**语法出错，可以在词法分析或者语法分析阶段检测**
2. 语义出错：源代码中，不符合语义规则的错误，有的可以在语义分析阶段得出，有的只能在运行时检测到出错。通常包括：类型不匹配、作用域错误 ...

# 书籍阅读

> 1. 编译器的定义

是一种语言的处理器，将一种语言翻译为另外一种语言

![[编译原理(龙书) 第二版.pdf#page=17&rect=397,755,455,822|编译原理(龙书) 第二版, p.17]]

> 2. 解释器的定义

按照用户的输入，执行源程序中的操作，得到结果

> 3. 预处理器的作用

将程序中多个位于独立文件的模块聚合起来，并将宏进行展开

> 4. 编译步骤

1. 词法分析
	将源程序的字节流，组织为有意义的 词素 的序列，对于每一个词素，产生词法单元（token）作为输出
	\<token-name, attribute-value>
	- token-name 用于在语法分析中使用的抽象符号
	- arrtibute-value 用于指向符号表中这个词法单元的条目
	符号表中的条目包含标识符的信息，比如名字和类型
	
2. 语法分析
	使用词法分析得到的词法单元，组织成树状的中间表示。这个中间表示就是词法单元的语法结构。语法树就是常用的方式，树的**每一个内部结点表示一个运算**，结点的子节点就是运算的分量
	
3. 语义分析
	利用语法树和符号表的信息，检查源程序是否和语言要求的语义一致。同时也负责收集类型信息，把信息放在语法树或者符号表中，用于中间代码生成。
	语义分析的一个重要部分是类型检查，检查每一个运算符有没有的匹配的分量，比如数组下标运算的运算符必须是整数。
	
4. 中间代码生成
	在把源代码翻译为目标代码的过程中，一个编译器可能构造出很多的中间表示。语法树就是一种中间表示，用于语法分析和语义分析。
	在语义分析完成后，很多编译器会先生成一个低级表示的程序，用来帮助翻译为机器语言。
	
5. 代码优化
	机器无关的优化步骤，目的是为了生成更好的目标代码，更好的定义往往是运行更快或者能耗更低。
	比如我们常写的使用自动类型转换的运算，float a = b * 60， 这个运算在语义分析阶段，收集到a的类型信息和运算结点的分量60，在中间代码生成阶段，会专门把60转为浮点数类型。
	代码优化的工作，会把源代码的60，直接替换为60.0，所以中间代码的生成步骤中，可以消除掉60的类型转换语句。
	在比如字符串的拼接，如果都是常量的拼接，在代码优化阶段，就可以进行常量折叠，在java的常量池中，生成最终的目标值
	
6. 代码生成
	代码生成以中间标识作为输入，目标语言作为输出。如果是机器语言，就需要先输出汇编语言，然后进行指令集的翻译，这个步骤不复杂，重点是输出汇编语言。
	输出汇编语言，需要预先指定变量的寄存器位置或者内存位置。所以一个关键点就是合理安排变量存放的寄存器，我们之前学习的寄存器，也是按照作用来分类的。
	
7. 符号表管理
	符号表条目包括 token-name 变量名 作用域 类型等等
	编译器在编译过程中，要求能快速访问符号表，获取所需信息

将上述的步骤组织成趟：源代码 -> 中间表示 -> 目标代码
