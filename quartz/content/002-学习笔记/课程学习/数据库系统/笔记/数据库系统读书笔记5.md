> 数据库的完整性包括了数据库数据的正确性和相容性
> 其中，正确性指的是数据符合现实世界的语义和情况
> 相容性指同一对象在不同关系表中的数据相同、一致

## 第五章 数据库完整性

### 5.1 数据库完整性概述

数据库的完整性和数据库的安全性往往联系密切。
完整性保证数据库内的数据符合现实语义且正确，而安全性保证数据不被恶意篡改。
为了实现数据库完整性，DBMS 必须实现：
1. 提供定义完整性约束的机制
2. 提供检查完整性约束的方法
3. 提供完整性的违约处理方法
	如拒绝或者级联

### 5.2 实体完整性

#### 定义实体完整性

sql提供 `primary key`,来定义关系的主码。
	1. 对于主码包含一个属性类，可以定义列级约束或者表级约束
	```
	列级定义主键
	create table student
	(SNO int primary key)
	---
	表级定义主键
	create table student
	(SNO int
	primary key(SNO)
	)
	```
	2. 对于主码包含一个属性组，只能定义表级约束
	```
	create table sc
	(Sno char(8),
	Cno cahr(8),
	primary key(Sno, Cno)
	)
	```

#### 实体完整性检查和处理违约

1. 检查数据的主键值是否存在
2. 检查值是否为NULL

检查数据的方式可以是全表扫描（耗时），一种是利用B+树索引


### 5.3 参照完整性

#### 定义参照完整性

使用`foreign key`定义外码，用`references`指出被参考表的主码
	```
	create table sc
	(Sno char(8),
	Cno cahr(8),
	primary key(Sno, Cno)
	FOREIGN KEY(Sno) REFERENCES Student(Sno)
	)
	```
#### 参照完整性检查和处理违约

1. 检查外码数据值在被参考列中是否存在
2. 如果不存在，检查是否为null

涉及到插入数据、修改数据、删除数据，都有可能违约![[破坏参照完整性的处理方案.png]]
**违约处理中**：
	1. **拒绝**：顾名思义，拒绝操作，**这是默认的**
	2. 级联删除/更新：把造成参照表对应不上的元组，都删除/更新
	3. 设置为空值：把造成参照表对应不上的元组，外码值设置为空
**设置策略**
	```
	FOREIGN KEY(Sno) REFERENCES Student(Sid)
	ON DELETE/UPDATE CASCADE/NO ACTION
	```

### 5.4 用户自定义完整性

1. NOT NULL
2. UNIQUE
3. CHECK
4. DEFAULT

其中，对于check的完整性约束，还可以设置为元组级的约束（其实就是对多列进行一个组合的条件判断）

```
CHECK(Sex=‘女’ or Sname NOT LIKE 'MS.%')
```

### 5.5 完整性约束命名子语句

如果要给约束命名，SQL提供
```
contraint 约束名 约束定义
```
