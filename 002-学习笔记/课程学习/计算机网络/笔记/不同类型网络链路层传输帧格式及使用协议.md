# VLAN 虚拟局域网

## 虚拟局域网帧格式

虚拟局域网的帧是在MAC中类型2字节前加入了4字节的VLAN字段，前面12位好像是用来标志是否在虚拟局域网中发送消息，交换机中的地址表知道每一个接入适配器的虚拟地址，如果没有找到的话，会转发给其他交换机，最终交付给目的站点的时候，会把这4个字节给去掉

# WLAN

## 无线局域网连接的建立方式

无线局域网的基本通信方式，每一个站点需要先和基本服务集BSS中的一个接入点AP建立连接。
	连接方式：
		1. 被动扫描，AP周期性发出信标帧，移动结点等待信标帧，发送关联请求帧
		2. 主动扫描：移动节点主动发出探测请求帧，所有的AP发探测响应帧，结点选择一个发出关联请求帧，AP应答关联相应帧

## WLAN的接入控制方式

WLAN的接入控制方式决定了 BSS 中移动结点什么时候能够发送数据或者接受数据

### 点协调功能 PCF

- **工作方式**：集中控制、轮询式。
- **协调者**：AP（接入点）作为“点协调器（PC, Point Coordinator）”，负责整个网络的调度。
- **操作流程**：
    1. AP会定期进入PCF周期，在这个周期里，只有AP可以分配信道使用权。
    2. AP依次轮询每个STA（站点），问：“你有需要发送的数据吗？”
    3. 被轮询到的STA如果有数据，就可以发送；没有数据就跳过。
    4. 轮询结束后，信道控制权可以回到DCF模式，大家自由竞争。
- **优点**：避免冲突，适合对时延敏感的业务（比如语音、视频等实时业务）。
- **缺点**：实现和管理较复杂，实际应用较少，**主流WLAN大多采用DCF**。

### 分布协调功能 DCF

- **原理**：站点之间竞争信道，谁先检测到信道空闲、等待一段时间后（DIFS+随机退避），就可以发送数据。
- **特点**：去中心化，适合数据突发、不规则、实时性要求不高的环境。
- **机制**：类似于以太网的CSMA/CA（带冲突避免的载波监听多路访问）。

	**CSMA/CA:**
	DCF的核心机制，主要通过“尽量避免冲突的发生”，而不是“发生冲突后再检测和处理”。
	- **机制**：
	    - 发送前先监听信道是否空闲（CSMA/CA）。
	    - 若空闲，等待一段随机退避时间（避免多个节点同时发）。
	    - 若不空闲，继续等待。
	    - 采用RTS/CTS（请求发送/清除发送）机制进一步减少冲突，尤其是针对“隐藏节点”问题。
	    - 发送完成后等待ACK（确认帧），若未收到ACK则重发。
	- **优点**：不需要“边发边听”，更适合无线环境。
	>为什么不使用 CSMA/CD 
	>在WLAN（例如Wi-Fi）环境下，物理特性和有线网不一样：
	>1. **发送端无法检测冲突**：无线设备发射信号时，自己的信号非常强，根本收不到别的终端的信号，所以无法像有线网那样“边发边听”来检测冲突。
	2. **隐藏节点问题**：有些终端之间互相听不到对方（比如A和C距离远），但都能和AP通信。这时A和C可能会同时向AP发数据，发生碰撞，但A和C自己都不知道。
	3. **信号衰减和干扰大**：无线信号受距离、障碍物影响大，冲突检测更加困难。

### 竞争窗口（Contention Window, CW）

- **定义**：竞争窗口是802.11 DCF机制下，为避免多个站点同时发送导致冲突而设定的一个随机退避时间范围。
- **机制**：
  - 每个想发送数据的站点，在等待DIFS后，需随机选择一个退避时间，单位称为时隙（Slot）。
  - 起始窗口为CWmin，发生冲突后窗口范围指数递增，最大不超过CWmax。
  - 退避计数器在信道空闲时递减，信道忙则暂停，减到0时才发送数据。
  - 若发生冲突，窗口加倍（Binary Exponential Backoff），提升公平性和抗冲突能力。
- **意义**：通过退避机制，大大降低了冲突概率，尤其在高并发场景下效果显著。

### 802.11帧结构（Frame Structure）

#### 1. MAC帧的基本结构

| 字节数    | 字段名 | 主要作用                  |
| ------ | --- | --------------------- |
| 2      | 帧控制 | 标识帧类型、功能、控制信息（见下详细分解） |
| 2      | 持续期 | 通信时长或NAV值，用于信道预约      |
| 6      | 地址1 | 一般为接收方MAC地址或BSSID     |
| 6      | 地址2 | 一般为发送方MAC地址           |
| 6      | 地址3 | 依网络类型不同，用于目的/源/BSSID等 |
| 2      | 序号  | 表示帧序列号与分片编号           |
| 6（可选）  | 地址4 | 仅WDS等特殊场景使用           |
| 0~2312 | 数据  | 实际传输的数据载荷             |
| 4      | FCS | 帧校验序列，检错用             |

#### 2. 帧控制字段详细分解（2字节 = 16位）

| 位数  | 名称   | 作用说明                                     |
| --- | ---- | ---------------------------------------- |
| 2   | 协议版本 | 目前固定为00，表明802.11标准版本                     |
| 2   | 类型   | 指明帧类别（00管理帧，01控制帧，10数据帧）                 |
| 4   | 子类型  | 精细区分帧类型（如Probe、Beacon、ACK、RTS、CTS等）      |
| 1   | 到DS  | 指示帧是否要发往分布式系统（AP/有线段）                    |
| 1   | 从DS  | 指示帧是否来自分布式系统                             |
| 1   | 更多分片 | 数据被分片时，除最后一片外均置1                         |
| 1   | 重试   | 重发帧该位为1，接收端可丢弃重复帧                        |
| 1   | 功率管理 | 指示发送端当前功率管理模式                            |
| 1   | 更多数据 | AP通知终端有未取走的数据                            |
| 1   | WEP  | 是否加密（Wired Equivalent Privacy，已被WPA/2取代） |
| 1   | 顺序   | 顺序相关，极少用                                 |

#### 3. 地址字段说明

- 数据帧最多可含4个地址，分别用于接收方、发送方、BSSID（AP的MAC）、转发地址等。
- 管理帧和控制帧地址字段较少，按功能需求精简。

#### 4. 序号字段（2字节）

- 高12位为帧序号，低4位为分片编号。
- 有助于数据重组与丢包重发。

#### 5. FCS校验

- 使用CRC-32算法，对前面所有字段进行校验，接收方检测数据包是否出错。

#### 6. 数据字段

- 变长，最大2312字节。数据帧则为上层数据载荷，管理帧/控制帧为相关控制信息。

### 分片发送（Fragmentation）

- **目的**：无线信道误码率高，大包丢失重发代价大，因此将大包分片发送，每片单独确认。
- **过程**：
  1. 发送端将大包分割成若干小片，每片封装成独立802.11帧。
  2. 发送首片需退避+DIFS，后续分片用SIFS快速发送。
  3. 每片都需ACK，如丢失仅重传该片。
  4. 帧头“更多分片”指示后续是否还有分片。
- **优势**：显著降低因差错导致的重传开销，提高整体吞吐率和可靠性。

---

### 信道预约（NAV, Network Allocation Vector）

- **定义**：“持续期”字段指明本次通信占用信道的总时间，其他站点据此设置NAV定时器。
- **机制**：
  - 站点收到带有持续期信息的帧后，将NAV设置为指定时长，在此期间不再争用信道。
  - **RTS/CTS机制**：发送端先发RTS，接收端回CTS，双方广播预约信道时间，防止隐藏节点干扰。
- **意义**：实现虚拟载波监听，提升多用户环境下的信道利用率，减少冲突。

---

### 站点切换（Roaming）

- **定义**：无线终端在多个AP覆盖区移动时，为保证持续网络连接，动态切换到信号更强的新AP。
- **过程**：
  1. 终端定期扫描周边AP信号并评估质量。
  2. 当前AP信号衰减或链路质量变差时，选择并关联到新AP。
  3. 需进行身份认证与关联过程，期间可能短暂断连。
  4. 若支持802.11r等快速切换标准，可显著减少切换延迟、提升体验。
- **应用**：企业WLAN、大型校园网、商场等多AP场景下尤为重要。

### 补充知识点

- **帧类型及用途示例**：
  - 管理帧：Beacon（广播）、Probe（探测）、Authentication（认证）、Association（关联）
  - 控制帧：RTS、CTS、ACK
  - 数据帧：实际数据传送
- **优先级机制**：SIFS < PIFS < DIFS（数值越小优先级越高，SIFS用于ACK/CTS/分片等关键控制帧）
- **安全机制**：WEP已不安全，现代WLAN普遍采用WPA2/WPA3进行数据加密和身份认证。
