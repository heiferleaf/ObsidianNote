# 网络基础知识笔记

---

## 一、主机的IP地址与 MAC 地址

### 1. IP地址
- **定义**：网络层地址，用于在整个网络中唯一标识主机。
- **类型**：IPv4（如 192.168.1.1）、IPv6（如 2001:db8::1）
- **作用**：实现跨网络寻址与路由，是数据包在网络层的目标定位。

### 2. MAC地址
- **定义**：数据链路层地址，网卡的唯一物理地址，通常由制造商烧录。
- **格式**：6字节（48位），如 `00:1A:2B:3C:4D:5E`
- **作用**：在局域网内实现主机识别和数据定向传输。

---

## 二、网卡（Network Interface Card, NIC）

### 1. 网卡的内部结构
- **MAC控制器**：负责帧的组装、解析、MAC地址识别、CSMA/CD等机制。
- **物理层接口**：实现物理信号的收发（如RJ-45以太网接口，光纤接口等）。
- **缓冲区**：用于临时存储接收或待发送的数据。
- **协议处理器**：实现部分数据链路层和物理层协议。
- **驱动程序接口**：与操作系统驱动程序通信，实现数据交互。

### 2. 每部分的作用
- **MAC控制器**：帧封装/解封装，地址过滤，冲突检测。
- **物理层接口**：信号调制/解调，实现物理传输。
- **缓冲区**：减少丢包，提高收发效率。
- **协议处理器**：执行协议相关任务（如CRC校验）。
- **驱动接口**：为系统提供统一访问网卡的能力。

---

## 三、以太网通信方式

### 1. 通信方式
- **早期总线型**：所有设备共享同一电缆，采用CSMA/CD冲突检测。
- **集线器型**：数码广播，冲突域大，所有端口收到相同数据。
- **交换机型**：智能转发，独立冲突域，点对点全双工通信更高效。

### 2. 通信流程
- 源主机通过网卡将数据封装为以太网帧，指定目标MAC地址。
- 数据通过物理介质发送至集线器/交换机。
- 交换机根据目标MAC地址选择性转发数据至目标端口，实现定向通信。

---

## 四、以太网帧结构

以太网帧结构如下：

| 字段          | 长度（字节）  | 说明                 |
| ----------- | ------- | ------------------ |
| 前导码         | 7       | 帧开始信号，同步用（物理层加上去的） |
| SFD         | 1       | 帧起始定界符（物理层加上去的）    |
| 目标MAC地址     | 6       | 接收方网卡地址            |
| 源MAC地址      | 6       | 发送方网卡地址            |
| 类型/长度       | 2       | 上层协议类型或数据长度        |
| 数据（Payload） | 46~1500 | 具体传输的数据            |
| 填充          | 0~（可选）  | 数据不足时补齐最小长度        |
| CRC(FCS)    | 4       | 循环冗余校验，错误检测        |

- **CRC校验范围**：从目标MAC地址到数据/填充部分，不包括前导码、SFD和CRC字段本身。

## 五、总线-集线器-交换机-网卡的联系

### 1. 连接关系
- 网卡是主机与网络的接口。
- 网卡通过网线连接到集线器或交换机。
- 交换机/集线器将所有连接的设备组成局域网。
- 早期网络采用总线型连接，现代多用交换机连接。

### 2. 技术演变
- 总线型拓扑 → 集线器（模拟总线） → 交换机（智能分发）

### 3. 数据流动
- 主机通过网卡以太网帧发送数据，交换机根据MAC地址智能转发至目标主机。

### 4. 关系图示

```
[主机A]---网卡---\
[主机B]---网卡---+---[交换机/集线器]---主干网线/总线---[其它设备]
[主机C]---网卡---/
```

---

## 六、总结

- **IP地址**用于网络层寻址，MAC地址用于链路层识别。
- **网卡**实现物理和链路层协议，是主机接入网络的桥梁。
- **以太网**采用帧结构和MAC地址实现高效局域网通信。
- **总线到交换机的演变**提升了网络效率与可靠性。
- **交换机和网卡**是现代局域网最核心的硬件基础。
