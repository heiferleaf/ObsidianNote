# 技术选型

1. 前端：ArkTs
2. 后端：Python+鸿蒙的云函数
3. 数据库：使用本地数据库

# 应用明确

## 一、应用定位

- **定位名称**：智能随行管家 / AI成长日记
- **核心目标**：帮助用户自动整理和总结每天的丰富活动、体验与成长内容。用户只需简要输入（文本/图片/语音转写），由AI自动生成风格多样、内容丰富的日记和成长反思，让个人成长有据可循，生活精彩不遗漏。
- **适用人群**：学生、在职个人、自我驱动成长者、忙碌但希望记录和提升自我的普通用户。
- **产品调性**：智能、温暖、主动陪伴、低干扰、支持自定义风格和个性化成长激励。

---

## 二、应用功能总结

1. **多模态内容采集**
   - 支持文本输入（简要记录每段活动/心情/事件）。
   - 支持语音转写输入（方便碎片化记录，后台转文字）。
   - 支持图片选择（上传当天相关图片佐证或丰富体验）。

2. **智能内容整合与AI分析**
   - 按日批量整理当天所有素材，AI分析关联事件/主题。
   - AI生成完整日记/报告，风格可选（活泼、精简、成长反思、安慰鸡汤等）。
   - 自动提取标签、关键词、事件总结。
   - 针对“能力成长/问题反思/情感安慰”场景，AI可输出针对性的成长建议或温暖话语。

3. **记录管理与浏览**
   - 本地保存日记、成长报告，历史回溯与检索。
   - 支持按日期、标签、风格等多维度查阅过往成长档案。

4. **总结&复盘功能**
   - 支持每日总结一键生成，可以手动触发，也支持定时提醒。
   - 汇总并展示用户一天的高光时刻、成长轨迹、反思与新目标建议。

5. **界面与体验**
   - 简约美观的录入页、多风格日记展示，卡片式汇总。
   - 容易上手、低学习曲线

---

## 三、技术实现总结

1. **前端：HarmonyOS ArkTS实现**
   - 使用ArkTS进行页面开发与业务逻辑编排。
   - 利用系统API实现文本/图片输入、HTTP网络请求、提示/弹窗、数据本地持久化。
   - 核心功能页面包括素材收集页、日记生成页、历史浏览页等。
   - 本地数据库使用 ArkTS Relational Store API，所有日记和成长数据完全存于用户设备。

2. **后端：Python云函数（Serverless）**
   - 在华为云等Serverless平台用Python实现后端AI整合服务。
   - 接收用户素材，调用各类AI开放平台API（如文本分析、情感识别、文风生成、成长建议），返回结构化报告给前端。
   - 免维护服务器，免费额度可满足日常需求，无须手工运维。

3. **AI能力集成**
   - 调用华为云、百度智能云、讯飞等NLP/AI开放接口，实现语音转写、情感分析、自动摘要、风格转换、能力建议等。
   - 前端和后端通过HTTP/HTTPS协议安全通信。

4. **上架与合规**
   - 方案规避摄像头等硬件依赖，所有功能模拟器即可测试，保证开发与上架平滑无障碍。
   - 本地数据存储保证隐私，云端API仅用于AI分析（可只上传必要内容），合规性强。

# 需求用例

## 1. 基本功能

### 1.1 多模态日常素材采集
- 文本录入：支持用户手动输入每一条经历简述。
- 语音转写：支持录音上传，后台AI接口转写为文本并保存（模拟器可用云API转文字）。
- 图片上传：支持从相册选择（非拍照）图片，作为当天事件补充（模拟器支持）。

### 1.2 智能内容分析与汇总
- 自动提取：对当天所有素材（文本/语音/图片）自动识别主题、情绪、关键词。
- 智能摘要：AI自动生成多风格精选日记总结（如幽默/成长/精简/温馨）。
- 成长建议：根据今日经历、AI分析，智能生成个人能力提升建议（如学习计划/问题改进方案/情绪疏导）。

### 1.3 历史记录与数据管理
- 本地数据库永久存档所有日记、AI分析结果及成长摘要。
- 按日期/标签/类别筛选查阅历史日记。
- 支持更新、删除任意历史记录。
- 支持导出日记文本为txt/md/pdf（选做）。

### 1.4 交互与界面
- 简洁的多标签主界面：首页、采集页、总结页、历史页等。
- 日记风格一键切换（如精简、活泼、励志等）。
- 晚间/指定时段定时“生成今日总结”提醒功能（或手动生成）。
- 记录详情页支持素材回看、AI分析、二次编辑。

### 1.5 AI/云端能力
- 调用后端Python云函数，集成AI云API(文本摘要、情感分析、风格生成、成长建议)。
- 支持批量上传当日所有素材，AI自动整合并返回内容。
- 语音转写结果与AI分析同步存储在本地数据库。

### 1.6 权限与合规性
- 仅申请必要的本地存储、网络通信、相册访问权限。
- 无需摄像头/定位/蓝牙/NFC等敏感权限。
- 明确隐私协议，所有数据本地保存，AI分析仅传必要内容。

---
## 2. 流程举例

1. 用户记录一天中多条体验，分别为文本、语音（自动转写）、图片若干。
2. 晚间一键生成或收到提醒后自动合成当日日记。  
   - AI分析各记录内容，合成结构化总结（事件、情感、成长点）。
   - 生成多风格文本供用户选择或修改保存。
3. 用户浏览当天成长总结、检索历史数据或导出、回顾AI建议。
4. 后端Python云函数处理AI整合任务，ArkTS端本地数据库管理个人数据。

---

## 3. 技术约束

- 所有功能需可于模拟器完整运行测试，无特殊硬件依赖。
- 本地数据用ArkTS官方Relational Store管理。
- 云端AI处理对接Python云函数，不依赖专有服务器，优先基于公有云API。
- UI需适配主流鸿蒙设备标准，无需分布式能力、复杂多端同步。

# 需求用例明细

## 1. 记录素材页面 - 一屏多入口

### 1.1 界面组织
- 采用**一屏多入口结构**，即“添加记录”页面同时呈现文本、语音、图片三类录入区域，用户可按需任选、随时添加。
    - 文本输入区：多行文本框，支持快速输入一段文字，配“保存”按钮。
    - 图片输入区：“选择图片”按钮（调用系统相册），支持多张图片，附加文本描述可选。
    - 语音输入区：“录音”按钮（后期真机可实现，模拟器阶段可选取音频文件模拟），后台自动语音转写为文本，转写内容可二次编辑，配“保存”按钮。
- 每种记录类型都可以一天多次添加，互不干扰。

### 1.2 数据管理
- 每条素材都作为独立数据项保存进本地数据库，字段包括类型（文本/图片/语音）、内容、时间戳、可选标签等。
- “今日素材”区域在首页或者专门的“今日素材”区下方，显示所有今日已录入条目（按添加时间排序）。
- 素材类型显示图标区分（如📝、🖼、🎤），列表支持点击查看/编辑/删除。
- 新添加的素材自动刷新进“今日素材”展示区。

### 1.3 交互体验
- 用户可在首页直接点击“添加新记录”快速进入素材录入区。
- 一天可不定次数添加多条素材，鼓励碎片化、随时记录。
- 素材保存后立即在哪都能看见（首页今日素材区）。

### 1.4 技术实现与约束
- 使用ArkTS前端+本地关系型数据库Relational Store管理所有素材数据，完全支持鸿蒙模拟器和上架。
- 录音模块模拟器测试阶段用上传本地音频/伪数据实现，后期真机可用官方录音接口。
- 图片仅支持相册选择，避免依赖相机硬件，保证模拟器和真机体验一致。

## 2. 日记生成选项页面

- 用户在首页点击「生成日记」按钮，跳转至“生成日记详细选项”页面。
- 在该页面自主配置本次生成日记的输出结构、风格与相关细节。

### 2.1 内容板块（内容格式区块，多选&可自定义）

- 显示系统预设板块（如：生活记录、学习内容、工作总结、情绪垃圾桶、明日计划等）。
- 用户可“添加新板块”自定义内容类型（每次添加后会记录在本地数据库，日后可复用、修改、删除）。
- 所有板块支持排序（如拖拽左侧顺序区）与移除。
- 长按任意板块，弹窗显示该板块详细功能与适用场景说明。

### 2.2 日记风格（表达形式，多选&可自定义）

- 系统内置多个表达风格（如：幽默风趣、客观理性、暖心细腻、简洁提要、激励鸡汤等）。
- 用户可自定义新风格保存本地，支持随时选择、移除、排序。
- 可同时选择多个风格，AI合成时按组合生成。

### 2.3 输出结构／格式相关选项

- **长度选择**：短/中/长（用户可选本次输出精炼还是详细）。
- **段落顺序**：页面展示已选内容板块顺序（用户可自定义拖拽调整，最终生成的内容遵循此顺序）。
- **分段/小标题**：可开关是否为每个板块自动添加小标题（如“学习收获”、“情绪分析”等）。
- **段落分隔符样式**：可选普通间隔/艺术花边/空行（提升可读性）。
- **展示方式**：支持纯文本、图文混排（自动插入今日图片素材，测试阶段也可单纯开关）。
- 其它结构参数（如每段加问候语、结尾加一句激励等，后续可拓展）。

### 2.4 附加补充需求说明

- 页面下方有输入框，用户可对本次AI生成额外说明，如“多关注情绪”、“用诗体风格”、“不要太长”等；
- 用户补充的说明内容会一起提交给AI生成接口，提升个性化。

> 交互设计亮点
>
>- 右侧为可选项池，左侧为已选内容区，用户通过“拖拽”将板块和风格拉入左侧并排序。
>- 板块与风格均支持新建、编辑、删除，操作简单直观。
>- 长按任何选项弹出详细说明，防止新手迷茫。
>- 当前配置支持“一键保存模板”，“下次一键复用”自己习惯的日记结构。
>
> 数据存储与复用
>
>- 用户自定义的内容板块、风格项均存储于本地数据库；
>- 历史配置可作为模板保存，下次生成时直接调用、修改。

## 3、AI生成日记内容的渲染方式

### 1. 基本渲染方案

- **渲染格式：Markdown**
    - AI生成的日记结果转换为标准Markdown格式（支持标题、分区、小标题、粗体、斜体、有序/无序列表、表格、分割线等）。
    - 每个内容板块自动以Markdown小标题（如`## 学习内容总结`）开头，正文采用段落文本、列表等合适样式。
    - 图片素材可通过`![图片描述](图片地址)`的Markdown语法内嵌，自动排序于合适位置。
    - 若AI结果仅为纯文本，由前端补充转换为基础Markdown结构。

- **鸿蒙ArkTS渲染实现建议：**
    - 前端用社区现有或自研Markdown渲染组件，将AI返回的Markdown字符串直接渲染为富文本界面，支持样式、分段、图文混排。
    - 若无完全集成Markdown解析器，也可将结构化数据分块转换为ArkUI原生组件集群（如Section/Card/Divider/Text/Image等）。

- **可编辑性：**
    - 日记详情页可提供“编辑模式”：自动切换为Markdown编辑器或富文本输入区，允许用户修改文本、版块结构、插图顺序等。

## 4、生成日记后的导航与返回逻辑

### 1. 生成日记后的跳转与详情页

- 用户在“生成日记详细选项页”配置后，点「生成日记」：
    - **直接跳转至“日记生成结果详情页”**（而非首页或历史页）
    - 详情页用于完整、清晰、一站式查看新生成的AI日记，可支持编辑、保存、复制、分享等操作

- **详情页顶部/显著位置有返回按钮**，返回行为明确：
    - 若是“刚刚生成结果”进入详情页，则**返回首页**，让用户回到主操作流
    - 若由历史记录页点击某条日记进入，则返回历史页（见下）

- **其它辅助操作建议：**
    - 在详情页下方，可配“去历史页查看归档/所有日记”快速按钮，便于继续管理
    - 支持编辑后保存，或一键删除当前日记

---

### 2. 历史日记页的功能与结构

- 设独立Tab栏入口“历史”，随时可访问。
- 历史页内容结构：
    - 按**日期逆序**列表展示所有生成的日记
    - 每条卡片含日期、板块标题摘要，预览主要内容前几行或字数
    - 支持输入关键词/标签进行搜索，快速定位过往日记

- **点击某条历史日记**：
    - 进入该日记详情页（渲染格式同AI新生成结果）
    - 可再次编辑、复制、分享，或删除
    - 详情页返回时**回到历史列表**

- **数据存储**：
    - 每篇日记内容与其生成参数（结构、风格等）持久存储在本地数据库
    - 支持本地全量浏览、增删改查
