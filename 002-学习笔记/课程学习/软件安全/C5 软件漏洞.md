# 软件漏洞

> 软件漏洞，也称脆弱性

## 缓冲区溢出漏洞
   
   向固定大小的缓冲区，写入超过其容量的数据，多余的数据会覆盖相邻的内存空间。
   
   一般攻击的内存对象是栈中函数返回地址、堆的块首管理结构。
   
   缓冲区溢出的本质原因，是没有对数据边界条件，或者函数指针引用进行边界检查。
   
   缓冲区溢出的种类：栈溢出、堆溢出、异常处理SEH结构溢出、单字节溢出

### 栈溢出

在做栈溢出的实验的时候，需要注意栈中存放的多字节的数据，比如返回地址、局部变量（不是char类型）、栈基址值，在x86架构下，都是==小端法==存放。

> [!NOTE] 小端法
> 
> 假设我们有一个 32 位4字节的int变量 `int a = 0x12345678`。以字节为单位进行拆分，分别为 `0x12 0x34 0x56 0x78`
> 其中 `0x12` 是其中的高位，`0x78` 是其中的低位。在小端法中，栈中的存放的方式如下：
> 从低地址 - 》 高地址
> `0x78 0x56 0x34 0x12`

### 堆溢出

通过获取两个堆块 A B，（A地址 < B地址），向A中写入数据量大于 （&B - &A ）的数据，可以覆盖B的堆区内容。

不止于此，我们可以设置一个空闲块的块首信息，修改其中的 FD （下一个空闲块地址）和 BD（上一个空闲块地址）。
只需要设置 FD + 一段偏移量（BD在块首中的偏移量，对应 node->pre==->next==）为指定地址，把 BD内容设置为指定值，就可以向任意地址写入任意值。
这种攻击叫做 **Dword shoot** 攻击

> 堆块分配的时候，都会让 node->pre->next = node->next


### SEH溢出

异常处理结构体 SEH，是Windows提供的异常处理机制，对应的内核的实现，给高级语言的异常处理使用。

每一个 SEH 结构体包含两个 DWORD 指针，异常处理函数句柄 和 SEH 链表指针。
前者意如其名，后者是所有的异常处理方法串联的关键。

异常处理的串联：
- SEH 结构体是放在线程的栈区中的
- 当线程初始化，分配线程栈区的时候，就在栈中安装一个SEH，作为默认异常处理
- 当某一个函数代码，写了try catch代码段的时候，会在函数栈中安装 SEH 结构体，设置对应的函数处理句柄，和指向上一个 SEH 地址的指针
- 线程还持有一个 TEB（线程环境块），用来指向最顶端（也就是最深层的函数栈，因为栈向低生长）

SEH 攻击。就是通过覆盖 SEH链表的入口地址，异常处理函数句柄或者链表指针，控制程序执行流程

### 单字节溢出

溢出的大小是一个字节。
依赖于函数栈中的第一个局部变量可以被溢出


## 格式化字符串漏洞

C / C++ 中，格式化输出的函数，匹配的都是可变参数
```c
printf(format_string, arg1, arg2,..., argn);
```

| 格式符   | 输出类型/含义               | 说明/示例（简要）                            |
| ----- | --------------------- | ------------------------------------ |
| `%d`  | 输出整型（int），十进制         | 例：`printf("%d", 10);` → `10`         |
| `%ld` | 输出长整型（long），十进制       | 例：`printf("%ld", 100000L);`          |
| `%o`  | 以八进制形式输出整数            | 例：`printf("%o", 10);` → `12`         |
| `%x`  | 以十六进制形式输出整数（小写）       | 例：`printf("%x", 31);` → `1f`         |
| `%u`  | 以十进制输出无符号整型（unsigned） | 例：`printf("%u", 10u);`               |
| `%c`  | 输出单个字符                | 例：`printf("%c", 'A');` → `A`         |
| `%s`  | 输出字符串（char*）          | 例：`printf("%s", "hi");` → `hi`       |
| `%f`  | 输出浮点数（小数形式）           | 例：`printf("%f", 3.14);` → `3.140000` |

当我们指定格式化的字符串，但是没有传入参数的时候，就可以在函数调用栈中，向高地址逐步读取。

进一步，通过 %n 格式化，我们可以向任意地址写入想要的数据。

> [!NOTE]  %n
> 
> `printf("hello%n", &num);`
> 将输入的字符串的长度，写入到指定地址中。示例是把5（hello的长度）写到num变量的地址中，后续可以读取 num = 5。
> 
> 利用这个行为进行攻击，我们要让扫描数据的指针，指向传入参数的开头，通过 %p 测试偏移量，然后通过把字符串参数开头改为指定地址，长度改为打印出来的数据，通过 【地址】【补齐长度】【%偏移量】$n ，来直接向地址写入长度。


## 整数溢出

2. 类型转换，大转小截断
3. 有符号的运算，符号位疏忽

整数溢出不会单独使用

## 权限类型漏洞


水平越权、垂直越权。
- 水平越权：相同级别用户之间可以相互访问、删除、修改
- 垂直越权
	- 向上越权：低级别用户可以直接高级别用户的操作
	- 向下越权：高级别用户可以访问低级别用户信息


# 漏洞利用

针对软件中已有的漏洞，针对漏洞的类型和特点采取相应的技术方案，进行尝试或者实质上的攻击。
> Exploit（利用）


shellcode，植入进程的代码，是一种漏洞利用的技术。
漏洞利用Expolit，包含了漏洞的触发，控制权限转移，shellcode执行的完整流程，能实现Expolit目标的有效载荷，叫做Payload。
> Expolit 利用漏洞的行为，shellcode 实际入侵进程的代码，Payload 包含 shellcode攻击代码、漏洞触发代码，程序流程转移的代码。
> 通过 Payload，最后执行 Payload 中的 shellcode，整个过程叫做 Expolit


shellcode 的编写，有一些注意点：
- ==一些特殊字符需要编码处理==：通过strcpy函数做缓冲区溢出的时候，碰到null会认为字符串读取结束，所以 shellcode 对一些特殊字符需要编码；在不同的平台上，字符集可能会有差异，所以也需要进行编码；此外，编码还可以绕过一些安全检测
- 函数的API定位麻烦，特别是系统调用的API，封装一层之后给用户使用的。所以要通过==动态方式获取api==

# 漏洞利用防护

因为缓冲区溢出，本质是因为高级语言没有对缓存区的写入进行边界检查，所以为了防范，操作系统采用了一些防范技术。下面都是 Windows 操作系统中常用的防范漏洞利用的技术。

1. **ASLR**：Address Space Layout Randomization 地址空间布局随机化。shellcode 的执行，往往依赖一些已有函数，比如系统调用，dll中的函数，通过可执行文件加载的时候，随机化一些地址，让 shellcode 失效。
	ASLR随机化的关键系统地址包括: PE文件(exe文件和dll文件)映像加载地址、堆栈基址、堆地址、PEB和TEB（Thread Environment Block，线程环境块）地址等
2. **GS Stack protection**：栈缓冲区溢出保护，在返回地址和新栈帧缓冲区之间设置一个随机数，函数返回值，检查随机数是否被修改
3. **数据执行保护 DEP** ：设置堆栈内存代码不可执行，防止溢出之后的代码执行
	1. 分为软件 DEP 和硬件 DEP
4. **SafeSEH**：保护 SEH 结构体不被破坏
	1. 把所有的 异常处理函数 地址解析成一张异常处理表，表放在PE文件的数据块中
	2. PE 文件加载时，对上述的表进行随机数加密。*将加密后的SEH函数表地址、模块的基址、模块的大小、合法SEH函数的个数等信息，放入ntdll.dll的SEHIndex结构中*
	3. 在可执行文件运行时，如果需要进行异常处理，会先对表进行解密，然后检查要执行的函数是不是在表里，以及要执行的异常处理函数句柄是不是在栈中。溢出防止覆盖SEH和把shellcode放在栈中
5. SEHOP
