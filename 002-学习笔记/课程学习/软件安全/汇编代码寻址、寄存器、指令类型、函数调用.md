## 寻址方式

> 寻址的含义：寻找操作数地址的方式

在汇编代码中，寻址涉及到的内容无非就是两个方面
- 操作数据不在内存中
- 操作数据在内容中

寻址方式可以在这个基础上分类
1. 不涉及到内存的寻址
	1. 立即数寻址：把一个立即数做当目标地址
	2. 寄存器寻址：把寄存器中的值当目标地址
	3. 隐含的寻址：有一些指令，默认使用特定寄存器的值作为运算分量
2. 涉及到内存的寻址（x86中，【】符号表示去内存中找）
	1. 直接寻址 【立即数】
	2. 间接寻址【寄存器】：读取寄存器中的值addr，去把内存addr处的值作为操作数
	3. 相对寻址【寄存器 + 立即数】：寄存器的值 + 立即数 = addr
	4. 基址加变址【EBX + ESI】：适合一维数组
	5. 比例变址寻址【EBX + ESI * 4 + 0x10】：适用于寻找类对象数组中某一个对象的某一个字段
```cpp
	struct {
		int field1;
		int field2;
	} S; // 每一个结构体8字节，刚好符合对齐要求
	S[] ss; // 一个结构体数组
	ss[i].field2; // ss + i * 8 + 4 对应关系 ss : EBX ; i : ESI ; 4 立即数偏移量
```

在32位机的时代，函数的调用，所需要的参数是通过栈来传递的，而到了64位机的时代，大部分的参数传递，可以通过寄存器来完成，如果数量多，会通过栈来传递

## 经典的寄存器

### 第一部分：核心寄存器角色（CPU 的变量）
*(注：以 32 位 `E` 开头为例，64 位只需将 `E` 替换为 `R`)*

#### 1. 通用数据四金刚（做运算、存临时数据）
*    **`EAX` (累加器)**：**绝对的核心。** 所有函数的**返回值**都在这里。也是乘除法默认的存放地。
*    **`ECX` (计数器)**：**循环的保姆。** `for` 循环的 `i`，或者操作字符串时的长度计数。
*    **`EDX` (数据寄存器)**：**EAX 的副手。** 经常和 EAX 拼在一起处理大数据，或作为临时草稿纸。
*    **`EBX` (基址寄存器)**：**保险箱。** 常存数组首地址。它是“非易失”的，函数内部用它前必须先 `PUSH` 备份。

#### 2. 指针与变址四护法（玩转内存与数组）
*    **`ESP` (栈顶指针)**：**动态游标。** 永远指向内存栈的最顶端，随 `PUSH`/`POP` 自动升降。
*    **`EBP` (栈底/基址指针)**：**定海神针。** 永远指向当前函数栈帧的底部。找局部变量 `[EBP-4]` 和找参数 `[EBP+8]` 全靠它。
*    **`ESI` (源变址)**：**搬砖起点。** 拷贝数据时，指向“从哪里复制”。(在 64 位中，`RSI` 充当第 2 个参数)
*    **`EDI` (目的变址)**：**搬砖终点。** 拷贝数据时，指向“复制到哪里”。(在 64 位中，`RDI` 充当第 1 个参数)

#### 3. 幕后大管家（控制程序运行）
*    **`EIP` (指令指针 / PC)**：**进度条。** 永远保存“下一条要执行的指令地址”。不能直接修改，只能用 `JMP`/`CALL` 改变。
*    **`EFLAGS` (标志寄存器)**：**状态灯。** 存着上一条指令运算的结果（是否为 0？是否负数？是否溢出？），供后面的 `if...else` (跳转指令) 使用。

*(💡 附 64 位传参规则：前 6 个参数依次放入 `RDI, RSI, RDX, RCX, R8, R9`，剩下的压入栈中。)*

### 第二部分：基础汇编指令速查（CPU 的动作）
*(注：采用 Intel 格式：`指令 目的地, 来源地`)*

#### 1. 数据搬运（赋值）
| 汇编指令 | 高级语言语义 | 极简解释 |
| :--- | :--- | :--- |
| `MOV A, B` | `A = B;` | 把 B 的值赋给 A（B 可以是寄存器、立即数或 `[内存]`）。 |
| `LEA A, [B+C]` | `A = &B + C;` | **取地址**（Load Effective Address）。只算地址本身，不取内存里的数据。也常被编译器用来做极速加法运算。 |
| `PUSH A` | `Stack.push(A);` | 把 A 压入栈顶（`ESP` 减 4/8）。常用来传参或备份寄存器。 |
| `POP A` | `A = Stack.pop();` | 把栈顶的值弹出来给 A（`ESP` 加 4/8）。 |
> 对于函数调用的栈，x86的实现方式是：栈从高到低
> 1. 调用者保存寄存器 （Call 之前的push操作）
> 2. 多的函数参数 （==32位机的时候，是没有通过寄存器来保存参数的==，都是在栈上传递 | Call 之前的push操作）
> 3. 返回地址 （Call 操作实现）
> 4. 旧的栈基址寄存器 （被调用函数执行）
> 5. 被调用者保存寄存器
> 6. 临时变量
> 这些栈中变量的获取方式，是通过栈基址寄存器加上偏移量获取，对于局部参数，是栈基址 - 立即数， 对于函数的参数，是栈基址 + 立即数
> 当 Call 指令调用的之前，就会执行编译器生成的 Push 指令，压入上述1、2，然后压入返回地址，被调用者函数负责保存旧的基址寄存器，存放被调用者保存寄存器，在 RET 返回之前，则会通过 POP 对应的恢复原来栈帧中的寄存器环境，栈基址寄存器，栈指针寄存器


> 调用者视角 (Caller)
> 
> 1. **保存寄存器**：压栈（Push）保存调用者认为调用后还需要使用的寄存器 （ESP -）
>     
> 2. **传递函数参数**：将多余的参数（通常从右向左）压栈 （ESP -）
>     
> 3. **Call 指令**：硬件自动将下一条指令的地址（返回地址）压栈 （ESP -）  
>     （注意：此时 Call 刚执行完，ESP 刚好指向“返回地址”，准备跨入被调用者的地盘）
>     
> 
> 被调用者视角 (Callee)  
> 4. **存放旧基址寄存器并建栈**：执行 push ebp 保存调用者的旧 EBP（ESP -）。随后执行 mov ebp, esp 建立新栈帧。（此时访问 [ebp] 的内容，得到的就是旧 EBP；访问 [ebp + 4] 就是返回地址）  
> 5. **存放临时变量**：为局部变量分配栈空间，如 sub esp, X（ESP - X）  
> 6. **准备返回（打扫战场）**：
> 
> - 执行 mov esp, ebp：将 ESP 一把拉回 EBP 的位置，等效于瞬间释放了所有临时变量。
>     
> - 执行 pop ebp：将栈顶的值（旧 EBP）弹回给 EBP 寄存器。因为是弹栈出栈，此时栈收缩 （**ESP + 4** 或 **ESP + 8**）。
>     
> - （注意：做完这步，旧 EBP 恢复了，且此时的 ESP 刚好完美地指向了当初压入的“返回地址”）
>     
> 
> 1. **RET 返回**：相当于隐式的 pop eip。将当前栈顶的“返回地址”弹入 PC（指令指针）寄存器。因为再次发生了弹栈，（**ESP + 4** 或 **ESP + 8**）。
>     
>     - （至此，代码不仅跳回了调用者，而且 ESP 也精准回到了调用者当初刚执行完 Call 之前的状态，完美衔接！）
>

#### 2. 算术与逻辑运算
| 汇编指令 | 高级语言语义 | 极简解释 |
| :--- | :--- | :--- |
| `ADD A, B` | `A += B;` | A 和 B 相加，结果存入 A。 |
| `SUB A, B` | `A -= B;` | A 减去 B，结果存入 A。 |
| `INC A` / `DEC A` | `A++;` / `A--;` | A 自增 1 / A 自减 1。 |
| `XOR A, A` | `A = 0;` | 异或。**这是汇编中最常见的高级清零手法**，比 `MOV A, 0` 执行更快，占字节更少。 |

#### 3. 控制流（if/else 与 循环）
汇编没有大括号，逻辑全靠 **“比较(`CMP`) + 跳转(`Jxx`)”** 组合拳。

*   **第一步：设置状态灯**
    *   `CMP A, B`：**比较**（其实内部在做 `A - B`，但不保存结果，只改变 `EFLAGS` 的状态灯，比如减完是 0 就亮 Zero Flag）。
*   **第二步：根据状态灯跳跃**
    *   `JE 标号` (Jump Equal)：如果相等 `(A == B)`，跳！ (也叫 `JZ` Jump Zero)
    *   `JNE 标号` (Jump Not Equal)：如果不相等 `(A != B)`，跳！ (也叫 `JNZ`)
    *   `JG 标号` (Jump Greater)：如果大于 `(A > B)`，跳！
    *   `JL 标号` (Jump Less)：如果小于 `(A < B)`，跳！
    *   `JMP 标号` (Jump)：**无条件跳转**（相当于 `goto`），直接跳过去。

#### 4. 函数调用（方法执行）
| 汇编指令 | 内部隐藏动作 | 极简解释 |
| :--- | :--- | :--- |
| `CALL func` | `PUSH 下一条地址`<br>`JMP func` | **调用函数。** 自动把当前进度（返回地址）压入栈中，然后跳到 `func` 去执行。 |
| `RET` | `POP EIP` | **函数返回。** （写在函数末尾）从栈顶弹出刚才存的返回地址，跳回去继续执行原来的代码。 |
