# Web 基础

## HTTP

超文本传输协议HTTP：网络分层体系中的应用层协议，用于浏览器和Web服务器之间传输数据使用。
之所以叫做超文本，是因为传输的数据除了文本，还包括图片、视频、音频等非文本资源。

## HTML

超文本标记语言，HTTP协议传输的主要对象，包含文本，脚本等资源，支持被浏览器渲染。HTML中的元素，都是完整的标签对

## PHP

专门用于Web通信场景的一种解释型语言，可以根据前端的请求，生成动态网页，还支持对于数据库的连接。
后续的Web渗透，也依赖php语言对于动态网页的生成中的注入行为。

## Web应用的十大安全威胁

1. SQL 注入
2. XSS 跨站脚本
3. 遭到破坏的身份认证和会话管理
	Session信息或者Cookie信息被劫持了
4. CRFS：伪造跨站请求
5. 不安全的对象引用
6. ...

# Web渗透

## 文件上传漏洞

文件上传漏洞的步骤：
- 文件上传：可以绕过浏览器和后端的检查，把文件存储在Web服务器的目录中
- 文件可以被解析执行：当文件时脚本文件的时候，后端需要能识别文件类型，执行代码
- 文件上传后，需要能通过某种方式找到文件存储路径，才能发出请求执行文件内代码

## XSS 跨站脚本注入

用户在网页中的输入，生成了动态的HTML代码，js执行其中的脚本，就是XSS跨站脚本注入。不用关心这个跨站什么意思

- 反射式跨站脚本：通过请求的URL生成恶意的脚本代码，PHP生成的动态HTML中就包含恶意的脚本代码返回给浏览器执行。
	当然这个恶意代码不可能用来攻击黑客自己。黑客可以通过各种方式让别人打开这个url，让别人的浏览器执行生成的恶意代码。
	因为js代码不会显示在前端，所以用户感知不到。
- 持久化跨站脚本：**反射式的依赖让自己传播URL，让别人打开**，持久化的思路是把恶意代码放到服务器内部存储，别的用户访问正常页面的时候，会返回恶意代码，从而达到攻击目的

## SQL 注入漏洞

SQL 注入指：将用户的输入内容，作为SQL命令的一部分进行执行。

首先需要先找到能进行SQL注入的地方：单引号法，在输入的串中加上一个单引号，可以==验证==会不会把用户的输入当作SQL代码部分执行

`SELECT * FROM articles WHERE id = '1';`
`SELECT * FROM articles WHERE id = '1'';` 单引号注入后，可以插入一段获取信息的代码，信息有时候会附带在报错信息中回显在前端。

有的时候，信息不在前端展示，这时候SQL盲注技术：
- 布尔盲注，根据注入语句中做二分判断，前端的响应结果200还是5xx，来验证判断是否正确
- 时间盲注：根据响应时间，决定注入的语句是否正确
	- `1’ and if(length(database())=1,sleep(5),1) #没有延迟`
	- `1’ and if(length(database())=4,sleep(5),1) # 明显延迟`
	就说明数据库名长度为4
