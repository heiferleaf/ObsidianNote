前面学习过对于一般软件进程的内存空间安排，这篇笔记继续延伸，在已知一个软件运行时的进程内存空间，如果获取得到对应各个部分的反汇编代码。

## PE 格式

软件都是以可执行文件的方式运行的，PE 格式，指的就是Win32平台下可执行文件中的数据结构。
- 这里的可执行文件，不只是 exe，还有 dll 等
- 可执行文件包含的内容，不只是二进制的机器代码（对应代码段），还有很多别的信息。当可执行文件被执行时，OS会按照PE格式的约定，把文件中的某一部分，放在内存中间的指定区域
	PE 中规定的可执行文件的数据结构，把内容分为若干个Session（数据节），不同的数据放在不同的Session中
	- rsrc：存放程序资源，比如图标、字体等
	- text：二进制机器代码，汇编产生
	- idata：可执行文件使用时，所需要的外部库函数和文件的信息
	- data：已初始化的全局和静态变量
	- 还有别的Session，比如edata（导入导出表），reloc等等

## 加壳

可执行文件在磁盘中的存储，可能是经过加壳的。
**加壳**：**可执行程序资源压缩**。是一种保护可执行文件的手段，用户不能直接查看到源代码，但是在内存中可以正常运行。

文件中自带一个解压程序，在可执行文件运行之间，先运行这个解压程序，从而把内存中的代码进行正确解读。

壳可以分类：
- 压缩壳：减少软件体积大小，和安全无关
- 加密壳：重点在于保护程序


## 虚拟内存

之前说的内存空间的布局，数据段、代码段、堆区、栈区等，都是一个进程看到自己的内存空间安排，也叫做虚拟内存、逻辑内存空间。对应物理内存上实际的空间，则叫做物理空间
- 虚拟地址：VA，程序在虚拟地址空间看到的地址
- 物理地址：PA，程序实际在物理空间中的地址
- 转载基址：Image Base一个可执行文件加载到内容，虚拟地址不是从0开始的，而是从基址开始的，比如exe是常见的 0x00400000， dll 是 0x10000000。这个地址叫做装载基址，可以改变的
- 相对虚拟地址：RVA, Image Base + RVA = VA


## 文件在磁盘中和加载到内存中的区别

磁盘中的存储单位是 0x200 字节为单位分配的，一个 Session 不足 0x200 字节大小，会进行填充。
内存中的数据页大小单位是0x1000字节，一个数据节，不足一个页，会进行填充



## 调试使用的软件

### LordPE

**PE分析、修改、脱壳软件**，可以查看可执行文件的PE格式，有哪些数据节，还可以修改

> [!NOTE] 输入函数地址表 IAT Import Address Table
> 我们知道，库函数分为静态库和动态库，静态库在文件链接的时候，就可以确定函数的内存位置，从而实现函数的调用。而对于动态库来说，则是通过可执行文件加载到内存的时候，依赖于IAT，动态的找到磁盘中的函数文件，加载到内存中。
> 所以静态库和动态库的本质区别是磁盘中是否有函数的代码，对应衍生到函数加载时机的区别
> 为了适应不同版本的dll库，以及OS版本，IAT就是封装地址细节，让可执行程序正确调用函数。
> IAT的使用流程，编译汇编的时候，IAT的内容无关紧要，就是指向导入表的地址；在加载到内存，执行动态连接的时候，会根据导入表中需要导入的函数，通过动态链接器，把函数加载到内存中，地址放到IAT里；运行的时候，IAT中函数真实地址的转换信息。
