### 3.1 SQL概述

#### 3.1.1 SQL的产生和发展

#### 3.1.2 SQL的特点

1. 功能综合、风格统一

#### 3.1.3 SQL的基本概念

SQL 支持数据库系统的[[数据库系统读书笔记1#1.3 数据库系统的三级模式结构|三级模式结构]]。而在关系型数据库中，关系有[[数据库系统读书笔记2#^ffa31f|三种类型]]。
其中模式对应于基本表，是对存储数据的逻辑结构和特征表示，用户看来就是二维表，外模式对应视图和部分基础表，内模式对应实际的存储数据的文件。
而 SQL 支持对视图和基本表进行查询和其他操作,一个或者多个基本表对应一个存储文件，一个表可以有若干索引，索引页存放在存储文件中。
存储文件表达实际数据的物理结构，组成了关系数据库的内模式，其中物理结构是由 DBMS 设计的。
视图是从基本表或者其他视图导出的表，不实际存储数据，所以数据库中只会存放视图的定义。

> SQL 有一个标准，但是不同 DBMS 系统在实现 SQL 时有部分差别，所以不是所有的 数据库管理系统的 SQL 都是一样的。

### 3.2 数据定义

在现代的数据库系统中，**一个数据库管理系统**可以建立多个**数据库**，**一个数据库**可以建立**多个模式**,**一个模式**下可以包含**多张表、视图和索引等数据库对象**

#### 3.2.1 模式的定义和删除

1. 定义模式

```
create schema [模式名] authorization <用户名> ；
```

- 模式名未指定时，隐含为用户名
- 要创建模式，需要使用命令的用户有数据库管理员的权限或者 由 DBA 授予的 create schema 权限
- 在创建模式时，就定义了一个命名空间，可以进一步定义其中的数据表、视图、定义权限
```
CREATE SCHEMA <模式名> AUTHORIZATION <用户名>
	[<表定义子句>|<视图定义子句>|<授权定义子句>];
```


2. 删除模式

```
drop schema <模式名> <cascade|restrict>
```
- cascade 模式为级联模式，会把模式中的数据库对象全部删除
- restrict 模式为限制模式，如果模式中数据库对象（表、视图），就拒绝执行该语句

1. 定义基本表


#### 3.2.3 索引的建立和删除

1. **建立索引**

   ```sql
   CREATE [UNIQUE] INDEX <索引名>
     ON <表名> (<列名1> [ASC|DESC], <列名2> [ASC|DESC], ...);
   ```
   - `UNIQUE` 关键字表示建立唯一索引，索引列的值不能重复。
   - 可为表的一个或多个的列创建索引。
   - 索引默认用于加快数据检索速度，特别是在WHERE、ORDER BY、JOIN等操作中。

2. **删除索引**

   ```sql
   DROP INDEX <索引名>;
   ```
   - 删除已存在的索引，注意部分DBMS可能需要加表名，如`DROP INDEX <表名>.<索引名>;`。

3. **查询索引**

   - 查询表的索引信息（以 PostgreSQL 为例）：
     ```sql
     \di           -- 列出所有索引
     \di <表名>*   -- 列出某表的所有索引
     ```
   - 或用 SQL 语句（通用）：
     ```sql
     SELECT * FROM pg_indexes WHERE tablename = '<表名>';
     ```

4. **修改索引**

   - 一般索引不能直接修改，如需变更需先删除再新建。

### 3.3 数据查询

#### 一般格式

```sql
SELECT [DISTINCT] <目标列表>
  FROM <表名或视图名> [别名] [, ...]
  [WHERE <条件表达式>]
  [GROUP BY <列名1> [, <列名2>, ...]]
  [HAVING <分组条件>]
  [ORDER BY <列名1> [ASC|DESC], ...];
```
- `DISTINCT` 用于去重
- `GROUP BY` 用于分组统计
- `HAVING` 对分组后的结果过滤
- `ORDER BY` 排序

---

#### 3.3.1 单表查询

1. **查询所有列/部分列**

   ```sql
   SELECT * FROM <表名>;
   SELECT 列1, 列2 FROM <表名>;
   ```

2. **WHERE子句——条件筛选**

   - 比较运算：`=, <>, <, >, <=, >=`
   - 逻辑运算：`AND, OR, NOT`
   - 模糊查询：`LIKE '%abc%'`
   - 范围查询：`BETWEEN a AND b`
   - 集合查询：`IN (v1, v2, ...)`
   - 空值判断：`IS NULL`, `IS NOT NULL`

   ```sql
   SELECT * FROM student WHERE age >= 18 AND gender = 'F';
   SELECT * FROM course WHERE cname LIKE '%数据库%';
   ```

3. **去重查询**

   ```sql
   SELECT DISTINCT dept FROM student;
   ```

4. **表达式和函数**

   - 算术、字符串、日期等函数
   - 别名：`AS`

   ```sql
   SELECT name, age+1 AS next_year_age FROM student;
   ```

5. **排序**

   ```sql
   SELECT name, score FROM exam ORDER BY score DESC, name ASC;
   ```

---

#### 3.3.2 连接查询

1. **内连接（INNER JOIN）**

   ```sql
   SELECT s.name, c.cname
     FROM student s
     INNER JOIN sc ON s.sno = sc.sno
     INNER JOIN course c ON c.cno = sc.cno;
   ```

2. **外连接（LEFT/RIGHT/FULL OUTER JOIN）**

   ```sql
   SELECT s.name, c.cname
     FROM student s
     LEFT JOIN sc ON s.sno = sc.sno;
   ```

3. **交叉连接（CROSS JOIN）**
	就是笛卡尔积

   ```sql
   SELECT * FROM A CROSS JOIN B;
   ```

4. **自连接**
	需要起别名

   ```sql
   SELECT a.name, b.name
     FROM employee a, employee b
    WHERE a.manager_id = b.emp_id;
   ```

---

#### 3.3.3 嵌套查询（子查询）

1. **WHERE 子句中的子查询**
	标量子查询
   ```sql
   SELECT name FROM student
     WHERE dept = (SELECT dept FROM teacher WHERE tname = '张三');
   ```

2. **IN/ANY/ALL/EXISTS 语法**

   ```sql
   SELECT name FROM student
     WHERE sno IN (SELECT sno FROM sc WHERE score > 90);

   SELECT name FROM student s
     WHERE EXISTS (SELECT 1 FROM sc WHERE s.sno = sc.sno AND score > 90);
   ```

3. **FROM 子句中的子查询（派生表/临时表）**

   ```sql
   SELECT avg_score FROM (
     SELECT avg(score) AS avg_score FROM sc
   ) AS avg_t;
   ```

---

#### 3.3.4 集合查询

1. **UNION（并集，去重）**

   ```sql
   SELECT name FROM teacher
   UNION
   SELECT name FROM student;
   ```

2. **UNION ALL（并集，不去重）**

   ```sql
   SELECT name FROM teacher
   UNION ALL
   SELECT name FROM student;
   ```

3. **INTERSECT（交集）**

   ```sql
   SELECT sno FROM sc
   INTERSECT
   SELECT sno FROM award;
   ```

4. **EXCEPT（差集）**

   ```sql
   SELECT sno FROM sc
   EXCEPT
   SELECT sno FROM award;
   ```

---

#### 3.3.5 基于派生表的查询

- 派生表即临时表/子查询结果表，常用于复杂查询的分步实现。

  ```sql
  SELECT t1.sno, t1.avg_score
    FROM (SELECT sno, AVG(score) AS avg_score FROM sc GROUP BY sno) t1
   WHERE t1.avg_score > 90;
  ```

### 3.4 数据更新

#### 3.4.1 插入数据

```sql
INSERT INTO <表名> [(列1, 列2, ...)]
  VALUES (值1, 值2, ...);

-- 或插入查询结果
INSERT INTO <表名> [(列1, 列2, ...)]
  SELECT ... FROM ...;
```

#### 3.4.2 修改数据

```sql
UPDATE <表名>
  SET <列1> = <值1>, <列2> = <值2>
  WHERE <条件>;
```

#### 3.4.3 删除数据

```sql
DELETE FROM <表名> WHERE <条件>;
-- 删除所有行
DELETE FROM <表名>;
-- 或TRUNCATE TABLE <表名>; （更快但不可回滚）
```

# tips

1. 关系型数据库在使用视图的时候，会先找到视图的定义，然后转换为对基本代表的操作，这个转换过程叫做视图消解
2. 当基本表修改的时候，表和视图的映像关系有可能会被破坏，所以建议修改表后删除视图，在重建视图
3. 当视图只是去除了基本表的某些行和列，可称视图为**行列子集视图**
4. 当视图中的列在基本表中不存在，是表达式的时候，把表达式列叫做**虚拟列**，视图叫做带**表达式的视图**
5. 视图定义中的子查询使用聚簇函数或者分组的时候，视图叫做**分组视图**
6. 视图的作用
	1. 为不同类型用户提供不同视图，保证机密数据的安全性
	2. 视图提供了一定程度的逻辑独立性
	3. 简化用户操作
