## 学习目标

- 理解 miniOB 整体架构和模块划分
- 掌握各模块间的调用关系和数据流
- 深入理解核心模块的实现原理
- 能够独立分析和修改源码

## 第一周：基础架构理解（Day 1-7）

### Day 1-2: 环境搭建与整体认知

**学习内容：**

- 下载并编译 miniOB 源码
- 运行基本的 SQL 命令，观察程序行为
- 阅读 README 和官方文档

**实践任务：**

- 成功编译并运行 miniOB
- 执行简单的 CREATE TABLE、INSERT、SELECT 操作
- 使用 GDB 调试，设置断点观察程序执行流程

**理解重点：**

- miniOB 的设计目标和特点
- 项目目录结构和文件组织方式
- 编译系统和依赖关系

### Day 3-4: 整体架构分析

**学习内容：**

- 分析 `src/` 目录下的模块划分
- 理解 miniOB 的分层架构设计
- 梳理主要的头文件和接口定义

**核心模块识别：**

```
src/
├── observer/        # 主程序入口和框架
├── common/         # 公共工具和基础设施
├── sql/           # SQL 解析和执行
├── storage/       # 存储引擎
├── net/           # 网络通信
└── deps/          # 第三方依赖
```

**理解重点：**

- 各模块的职责边界
- 模块间的依赖关系
- 接口设计原则

### Day 5-7: SQL 执行流程追踪

**学习内容：**

- 从 `main()` 函数开始追踪 SQL 执行流程
- 理解请求处理的完整链路
- 分析关键的数据结构

**追踪路径：**

1. 网络请求接收 (`net/` 模块)
2. SQL 解析 (`sql/parser/` 模块)
3. 语义分析和优化 (`sql/optimizer/` 模块)
4. 执行引擎 (`sql/executor/` 模块)
5. 存储访问 (`storage/` 模块)

**实践任务：**

- 使用调试器完整追踪一次 SELECT 查询
- 画出 SQL 执行的调用栈图
- 识别关键的数据结构（如 Tuple、Table、Index 等）

## 第二周：SQL 解析与优化（Day 8-14）

### Day 8-10: SQL 解析器深入

**学习内容：**

- 词法分析器（Lexer）实现原理
- 语法分析器（Parser）的语法规则
- 抽象语法树（AST）的构建过程

**关键文件：**

- `src/sql/parser/lex.l` - 词法规则
- `src/sql/parser/gram.y` - 语法规则
- `src/sql/parser/parse_tree.h` - AST 节点定义

**实践任务：**

- 修改语法规则，添加一个简单的新语法支持
- 调试解析过程，观察 AST 构建
- 理解不同 SQL 语句的 AST 结构差异

### Day 11-12: 语义分析模块

**学习内容：**

- Resolver 模块的作用和实现
- 符号表管理和作用域处理
- 类型检查和语义验证

**关键概念：**

- 表和字段的元数据管理
- 表达式的类型推导
- 查询计划的初步构建

**实践任务：**

- 分析复杂查询的语义分析过程
- 理解错误处理和异常传播机制

### Day 13-14: 查询优化器

**学习内容：**

- 基于规则的优化（RBO）
- 查询计划的表示和转换
- 常见优化规则的实现

**优化技术：**

- 谓词下推（Predicate Pushdown）
- 投影下推（Projection Pushdown）
- 连接顺序优化
- 索引选择策略

**实践任务：**

- 对比优化前后的查询计划
- 分析优化器的决策过程

## 第三周：存储引擎深入（Day 15-21）

### Day 15-17: 缓冲池管理

**学习内容：**

- 缓冲池（Buffer Pool）的设计原理
- 页面替换算法（LRU 等）
- 脏页管理和持久化机制

**关键组件：**

- `BufferPoolManager` 类的实现
- `Page` 类的生命周期管理
- 并发控制和锁机制

**实践任务：**

- 追踪页面的加载和淘汰过程
- 分析并发访问的处理策略
- 测试不同负载下的缓冲池性能

### Day 18-19: 存储格式与记录管理

**学习内容：**

- 页面的物理布局设计
- 记录的存储格式（定长/变长）
- 空闲空间管理策略

**核心概念：**

- Slotted Page 结构
- 记录的插入、删除、更新操作
- 页面分裂和合并机制

**实践任务：**

- 使用十六进制编辑器查看数据文件
- 手动构造和解析记录格式
- 分析存储空间利用率

### Day 20-21: 索引系统

**学习内容：**

- B+ 树索引的实现原理
- 索引的创建、维护和查询过程
- 唯一索引和普通索引的区别

**重点内容：**

- B+ 树节点的分裂和合并
- 范围查询的实现
- 索引与表数据的一致性维护

**实践任务：**

- 分析 B+ 树的搜索路径
- 测试索引对查询性能的影响
- 理解索引维护的开销

## 第四周：执行引擎与高级特性（Day 22-30）

### Day 22-24: 执行引擎架构

**学习内容：**

- 火山模型（Volcano Model）的实现
- 算子（Operator）的设计模式
- 数据流的处理机制

**核心算子：**

- TableScanOperator - 表扫描
- FilterOperator - 过滤条件
- ProjectOperator - 投影
- JoinOperator - 连接操作

**实践任务：**

- 实现一个简单的自定义算子
- 分析算子间的数据传递过程
- 优化算子的执行性能

### Day 25-27: 事务与并发控制

**学习内容：**

- ACID 特性的实现机制
- 锁机制和死锁处理
- 事务日志和恢复机制

**关键概念：**

- 两阶段锁协议（2PL）
- 写前日志（WAL）
- 检查点机制

**实践任务：**

- 模拟并发事务的执行
- 分析死锁检测算法
- 测试崩溃恢复功能

### Day 28-30: 综合实践与总结

**学习内容：**

- 性能分析和调优技巧
- 代码规范和设计模式总结
- 扩展功能的设计思路

**综合项目：**

- 选择一个感兴趣的功能进行深入优化
- 编写技术文档总结学习成果
- 准备参加比赛的代码基础

## 学习方法建议

### 1. 代码阅读技巧

- 使用 IDE 的跳转功能，建立函数调用图
- 善用 grep、ag 等工具搜索相关代码
- 写注释帮助理解复杂逻辑

### 2. 调试技巧

- 熟练使用 GDB 设置条件断点
- 使用日志输出追踪执行流程
- 编写单元测试验证理解

### 3. 文档整理

- 每天总结学习要点，画出模块关系图
- 记录关键数据结构的定义和用途
- 整理常见问题和解决方案

### 4. 实践验证

- 修改代码验证理解的正确性
- 编写小型测试程序验证假设
- 对比不同实现方案的优劣
