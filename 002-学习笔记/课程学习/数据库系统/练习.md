## 前置说明

$$S:(Sno, Sname):学生表$$
$$C(Cno,CName)：课程表$$
$$SC(Sno,Cno,Grade): 联系表$$
## 1. 至少选了两门课的学生

```SQL
select S.Sname
from S
where S.Sno in (
	select Sc.Sno
	from SC
	group by SC.Sno
	having count(DISTINCT SC.Cno) >= 2
	)
```

### 2. 选修了所有课的学生姓名

(相关子查询的登峰造极)

```SQL
select S1.Sname
	from S as S1
	where not exists (
		select *
		from C
		where not exists(
			select *
			from SC
			where S1.Sno = SC.Sno and S1.Cno = C.Cno
		)
	)
```

### 3. 选修了“数据库”课程的学生

多表查询或者嵌套子查询都可以

多表连接 - 没有技术含量，就是连接在一起，where里写出Cname\==条件

嵌套子查询
	学生 - 根据找出的Sno，确定学生
		关联表 - 根据子查询的课程号作为条件，查找Sno
			课程表 - 查到课程的课程号

```SQL
select *
from S
where S.Sno in (
	select DISTINCT SNO
	from SC
	where Cno = (select Cno from C where CName = "shujvku")
)
```

### 4. 选课最多的学生姓名

```SQL
select *
from S
inner join (
	select SNO
	from SC
	group by SNO
	having count(*) = 
		select max(cnt)
		from (
			select count(*) cnt
			from SC
			group by SNO
		) as temp
) as t on t.SNO = S.Sno
```

## 简述视图的优点

1. 简化对数据的操作，便于使用SQL语句
2. 对隐私数据进行封装，提供数据库的安全性
3. 数据存储的逻辑结构变化时，通过调整视图的创建语句，保证应用程序使用数据的结构不变，实现逻辑独立性
4. 提供数据的不同视角，不同的用户对数据可以有不同的视角，更切近需求
