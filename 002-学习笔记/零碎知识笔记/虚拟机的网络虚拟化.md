> 本片笔记的目的是搞懂虚拟器在物理机上运行时，虚拟机之间的通信，虚拟机和物理机之间的通信，虚拟机和外部网络之间的通信，到底什么时候可行，需要做什么配置，数据传输的底层流程

## 写在前面

对于一款虚拟机管理软件，总是有配置虚拟网络的功能，其中往往涉及到对于虚拟网卡的配置。虚拟网卡，也是走通整个流程的核心。

什么是网卡？对于一个网络设备来说，比如我现在用的电脑，想要接入网络，和网络中的其他主机进行网络通信，交换数据，就需要在网络世界中，有一个唯一的标识（IP地址），想要访问互联网中的网站，总是要对域名进行解析（DNS服务器配置），为了让这些信息不需要我们手工配置，设置了一个可以分配这些信息给主机的服务器（DHCP服务器），而这些信息，都==保存在网卡中==。

可以说，网卡包含了进行网络通信所必须的信息。此外，网卡还具备物理层和数据链路层的功能，传输数据，封装成帧。

对于虚拟网卡，他没有真实网卡所具备的MAC硬件地址，不能像真实网卡一样，具备向传输介质收发数据。在所包含的信息层面，往往是用户配置的信息，在收发数据的层面，需要借助一个虚拟层和虚拟交换机
> 典型的数据发送流程：用户态 -> 内核态 -> 虚拟网卡VNIC -> 虚拟层 -> 虚拟交换机 -> (接受方)虚拟层 -> （接受方）虚拟网卡VNIC -> 内核态 -> 用户态

虚拟交换机，也就是在虚拟机的环境下，识别连接的虚拟网卡发送的数据包，并进行转发，是在虚拟层面进行数据转发的软件

## 三种网络设置

在 VMWare 的虚拟网络设置中，虚拟机有三种网络设置模式：桥接模式（VMnet0）、仅主机模式（VMnet1）、NAT模式（VMnet8）。
![[QQ_1773054807385.png | VMWare 的网络配置]]
其中名称这一栏，VMnet1、VMnet8 都是虚拟机管理软件配置的虚拟交换机。对于到我们的主机，在安装完 VMWare 之后，出现了虚拟网卡，显然，这是虚拟机管理系统，为了让主机和运行的虚拟机，进行网络通信，为我们准备的存放虚拟网段IP、DHCP等信息用的软件网卡。![[QQ_1773055205499.png | 主机上的网卡]]

可以发现，在主机的网卡中，并没有 VMnet0，也就是桥接模式下，主机没有对应的虚拟网卡。==简单说，这是因为在桥接模式下，虚拟机和主机处于相同的网段，所以不需要另外的虚拟网卡==，借助虚拟网桥，和虚拟机通信

下面就来详细介绍三种模式

### 一、 桥接模式

![[QQ_1773055465312.png]]

先看架构图，通过 VMWare 提供的虚拟交换机，将虚拟机的虚拟网卡，和主机的网卡，进行连接，实现了主机和虚拟机之间的相互通信。
而主机和虚拟交换机的连接，因为没有使用虚拟网卡，所以需要通过虚拟网桥，将物理网卡接收到的广播消息、组播消息、针对虚拟机的单播消息，交给虚拟交换机转发。

总结：
- 可行的通信对象：
	- 虚拟机和主机之间相互，通过虚拟交换机和虚拟网桥
	- 主机和外部网络，毫无疑问
	- 虚拟机和外部网络之间，通过虚拟交换机和虚拟网桥，最后借助主机网卡，完成通信，==这时候==，==物理机和一个交换机差不多==，转发同一个网段内的数据包
- 因为都处于同一个网段，所以虚拟机配置的DNS服务器IP，默认网关IP，子网掩码，需要和主机上的网卡配置一样

优势：
- 实现简单
缺点：
- 不适用于网络IP紧缺的环境，因为会占用局域网中的IP

### 二、NAT模式

![[QQ_1773059867630.png]]
原理核心是NAT（虚拟地址转换）技术。将虚拟机的IP地址、进程端口和物理机的IP地址和端口做映射，实现在公共网络中的通信。

==主机上的 VMnet8 虚拟网卡的作用是让自己有一个虚拟网段的IP，在通过虚拟交换机实现和虚拟机之间的通信==。
虚拟机和外部网络的通信，是 虚拟网卡 -> 虚拟交换机（发现目的MAC不是主机的虚拟网卡） -> NAT设备（IP和端口映射表）-> 主机的物理网卡（数据包的IP和port变为主机IP+暴露出来的映射端口）

当外部访问虚拟机时候，通过主机网卡后，转发给NAT设备（IP和端口转换），然后通过虚拟交换机，给对应的虚拟机

总结：
- 主机和虚拟机可以正常通信
- 主机和外部网可以通信
- 外部网和虚拟机可以通信，前提需要注册NAT转换的端口

好处：
- 节省了局域网中的IP资源
缺点：
- 需要自己配置虚拟机网络的DHCP服务器
- 配置NAT设备
- 需要通过注册端口映射，和物理机的防火墙入站设置，才能让外部网络和虚拟机通信

### 三、仅主机模式 Host-Only

![[QQ_1773060926152.png]]

仅主机模式和NAT模式几乎相同，只是缺少了NAT设备，这决定了：在不采取其他措施的情况下，虚拟机无法和外网通信。

当然，仅主机这个名字的含义，也暗示了虚拟机只需要和主机进行通信。如果想和外部网络通信，可以将主机网卡共享给主机上的 VMnet1 虚拟网卡

## 虚拟的网络配置命令


1. 查看虚拟机的IP地址
```
ip addr show
```

2. 查看虚拟机的默认网关IP地址
```
ip route show
```
> 在 NAT 模式下，默认网关的IP地址，就是NAT设备的IP地址，所以虚拟机访问外部啊网络，通过虚拟网卡-》虚拟交换机-》NAT设备-》物理网卡
> 在 桥接 模式下，默认网关的IP和主机的默认网关IP一致

3. 查看虚拟机的 DNS 配置
```
cat /root/etc/resolfv.conf
可以获取本地回环地址，也就是 dns 数据包，先通过回环地址，获取虚拟机真正的 dns 配置
resolvectl status
获取真正的 dns IP
```
> 在 NAT 模式下，真正的 dns 服务器，默认是NAT网关，说明通过 NAT 代理访问真正的外部 dns 服务器。也可以自己配置 dns IP和主机的dns配置一致


### NAT 模式下

- DNS 服务器正确配置（默认就是 NAT设备的IP）
- 网关正确配置 （也是NAT 设备的IP）
- DHCP 服务器正确配置，可以在虚拟机中设置由 dhcp 服务器配置虚拟机的网络信息
> 达成上述条件，可以在虚拟机内访问外网

- 在 VMWare 的虚拟网络编辑器中，设置 NAT 模式下的端口转发，比如 8080 ：8080，配置虚拟机的IP地址，指定协议类型比如 TCP
- 主机的防火墙 - 高级设置中 - 新建入站规则 - 选择协议类型 TCP， 允许的端口 安全访问。将主机的端口暴露给公网，从而可以通过浏览器访问
- 虚拟机内部开启一个监听8080端口的进程 `python3 -m http.server 8080` 这个进程可以访问虚拟机内的文件系统
> 完成上述设置，可以从外网访问虚拟机

这些都是在设置 dhcp 自动分配的前提下的操作
可以通过编辑 /etc/netplan/01-network-manager-all.yaml 文件。自己配置信息
```
sudo nano /etc/netplan/01-network-manager-all.yaml
```
配置示例
```
network:
  version: 2
  ethernets:
    ens33: # 虚拟机的虚拟网卡名
      dhcp4: false # false 为手动配置
      addresses: # 虚拟机IP
        - 192.168.126.100/24      # 自己指定，避开 .1 .2 和已有设备
      routes: # 默认网关
        - to: default
          via: 192.168.126.2      # NAT 网关，固定是 .2
      nameservers: # DNS
        addresses:
          - 192.168.126.2         # DNS 走 NAT 节点转发
          - 8.8.8.8               # 备用，这个是谷歌的DNS服务器
```

### 桥接模式下

桥接模式下，虚拟机和主机在同一个网段中，IP地址也是由主机的dhcp服务器配置的，网关，dns服务器，子网掩码都和主机一样。
虚拟机和主机的通信，借助虚拟网桥，虚拟机和外部网络的通信，通过虚拟网桥和物理网卡，物理网卡没有代理的作用，只起到转发流量的作用。

1. 设置虚拟机的网络适配器为桥接模式
	- 勾选复制主机的网络设置
2. 设计 VMWare 的网络适配器管理，为桥接模式，选择当前主机使用的物理网卡
3. 配置虚拟机的网络信息配置为 dhcp：true
	- /etc/netplan/01-network-manager-all.yaml
	- 
```
	network:
	  version: 2
	  ethernets:
	    ens33: # 虚拟机的虚拟网卡名
	      dhcp4: false # true 
```

使用上面的命令，就可发现虚拟的ip配置，网关配置，dns配置都变了。
此时虚拟机和主机、外部网络互通。
> 虚拟机ping主机是通过虚拟网桥到物理网卡接口，所以需要在主机内放行防火墙对ping的ICMP报文
```powershell
> netsh advfirewall firewall add rule name="Allow ICMPv4 In" protocol=icmpv4:8,any dir=in action=allow
```
