> 对于一个 TCP 连接，断开分为主动和被动的，主动的包含通过四次挥手断开连接；被动的包含RST标识、RTO多次超时重传、保活计时器多次超时
> 
> 这个知识可以引出一个有意思的问题：如果把物理网线拔了，TCP 连接会马上断开吗
这个问题在文章的最后回答
## 四次挥手

客户端：$ESTABLISH - {FIN\_WAIT1} - FIN\_WAIT2 - CLOSED$
服务端：$ESTABLISH - CLOSE\_WAIT - LAST\_ACK - CLOSED$

客户端到 CLOSED 之前需要等待两个 MSL 计时器时间
- 让本次连接中还在网络中的数据包失效
- 保证服务器端关闭（最后一次挥手信息没有到服务器会重传）

## RST标识

当通信的一方收到一个TCP报文段，其中的**RST**标识为1，会马上断开连接。这个断开连接不需要四次挥手。

发送 RST 标识的场景有：
- **端口不存在**：客户端尝试连接服务器的某个端口，但是这个端口在服务器中不存在，服务器向客户端发送 RST
- **进程意外结束**：当进程意外终止时，比如 `kill` 系统调用杀死进程，进程结束前，会给连接的对端发送 RST
- **收到了非法序号**：当连接中的数据包传输，发现其中的序号在错误范围内（不可能在当前窗口中能接收的序号），发送 RST
- **A已经关闭了，B还在发数据**：A 给 B 发送 RST

## RTO 超时次数过多

TCP 管理的计时器中包含了 RTO 计时器，当超时次数过多的时候，会主动关闭连接

## TCP 连接长时间空闲

TCP 管理的计时器中包含了 KeepAlive 保活计时器，当连接空闲过长时间，启动这个计时器，计时器连续超时次数过多，会关闭连接


## 回答问题

对于文章开始的问题，答案是不会，但是过段时间会断开。
物理网线影响物理链路层传输二进制数据，传输层是感知不到物理层的。所以不会马上断开连接。
拔网线之后，如果传输层传输数据，会因为RTO超时次数过多断开连接；如果传输层不传输数据，那么也会因为保活计时器超时次数过多而断开连接。
