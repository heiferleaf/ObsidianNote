# 计算机网络第一章：概述

> [!summary] 本章概览 本章涵盖网络基础概念、交换方式、性能指标和体系结构

---

## 一、基本概念

### 网络、互联网、因特网

- **网络**：由结点和连接结点的链路组成，结点之间可以数据通信和交换资源
- **互联网**：网络通过路由器连接，组成的更大的网络
- **因特网**：现在最大的互联网络

> [!tip] 概念辨析
> 
> - **internet (互联网)**：泛指各种网络协议组成的互联网络
> - **Internet (因特网)**：专指使用TCP/IP协议组成的互联网络

### 计算机网络的定义与分类

**定义**：通用、可编程硬件互连而成

**分类**：可按使用者、拓扑结构、覆盖范围、交换方式、传输介质等进行分类

---

## 二、计算机网络的组成

### 从工作方式视角

- **边缘部分**：连接互联网的主机，进行数据通信、资源共享。主要通信模式包括请求/响应和P2P对等
- **核心部分**：为边缘部分提供服务（连通性和数据交换），通过电路交换、报文交换、分组交换实现

### 从功能视角

- **通信子网**：网络体系的下三层（物理层、数据链路层、网络层）
- **资源子网**：网络体系的上层（传输层及以上）

> [!important] 核心功能 **数据通信**是计算机网络最基础、最重要的功能

---

## 三、交换方式

**交换**：指通过某种方式动态地分配通信资源

### 电路交换

**过程**：建立连接 → 通信 → 释放连接

**特点**：用户独占通信的物理线路

**优点**：传输延迟小、竞争小、实现简单、有序传输

**缺点**：线路利用率低、建立连接需要开销、灵活性差

**延迟计算**：

```
总延迟 = 建立连接延迟 + 传播延迟 + 源节点发送时间
```

### 分组交换

**过程**：把数据分为很多数据段，加上首部信息，组成分组。分组是传输的单位，分为虚电路和数据报方式

**优点**：线路利用率高、重传数据量小、对数据缓存要求小、无需建立连接、灵活

**缺点**：增加首部开销、数据报可能乱序到达/重复/丢失（需上层控制）、中间结点有发送延迟

**延迟计算**：

方法1（关注第一个分组）：

```
总时间 = 第一个分组到达终点时间 + (n-1)个分组的发送时间
```

方法2（关注最后一个分组）：

```
总时间 = 所有分组从源节点发送时间 + 最后一个分组发送后到目的节点的时间
```

### 报文交换

**过程**：以报文为单位，加上首部信息，在网络中传输

**优点**：线路利用率高、无需连接、灵活

**缺点**：缓存要求高、重传代价大、额外控制信息开销、中间结点有发送延迟

## 四、网络性能指标

### 速率

> [!warning] 单位换算易错点
> 
> - 操作系统中内存大小单位量级是 2¹⁰
> - 计算机网络中（如Kbps）量级是 10³

**单位换算**：

```
1 Byte(B) = 8 bit(b)
1 KB = 1000 B (十进制，网络传输)
1 KiB = 1024 B (二进制，存储)
```

### 其他性能指标

- **带宽**：表示网络的通信能力
- **延迟**：数据从源点到终点所需的时间
- **延迟带宽积**：传播延迟 × 带宽，表示链路上可以容纳的比特数
- **往返时间（RTT）**：从发送方发送数据到收到接收方确认的时间
- **吞吐量**：单位时间内通过某个网络的实际数据量
- **丢包率**：丢包主要原因包括差错校验检测出错丢弃包，以及拥塞导致路由器缓存满丢弃包

## 五、网络体系结构

### 为什么要分层

便于实现功能、便于更新、便于拓展

### 主要模型

- **OSI/ISO**：7层理论模型，未实际使用
- **TCP/IP**：4层，广泛使用
- **五层协议**：教学模型，结合两者优点

### 五层协议模型（自下而上）

1. **物理层**：在用什么信号传输比特数据
2. **数据链路层**：在链路上传输数据
3. **网络层**：在不同网络中传输数据
4. **运输层**：在主机的进程间通信
5. **应用层**：通过进程交互实现不同特定的网络应用功能

### 体系结构中的核心概念

- **实体**：能够发送和接收数据的硬件和软件
- **对等实体**：两个系统中同一层的实体
- **协议**：主机相同层之间为了通信定义的通信规则。==协议是水平的==
- **服务**：下层为上层提供的功能。==服务是垂直的==。协议是在通过上下层接口/使用下层服务的基础上实现的
- **服务访问点（SAP）**：上下层之间交换信息的接口
    - 数据链路层的SAP：帧的类型字段
    - 网络层的SAP：IP数据报中的协议字段
    - 运输层的SAP：端口字段

> [!danger] 不同模型的实现差异 ISO/OSI模型和TCP/IP模型的实现不同
> 
> **例：流量控制**
> 
> - ISO/OSI模型：数据链路层实现相邻节点间的流量控制
> - TCP/IP模型：传输层实现端到端的流量控制

## 本章重点与易错点

**核心概念**：

1. 理解网络、互联网、因特网的区别
2. 掌握三种交换方式的特点和延迟计算
3. 理解网络体系结构分层的意义
4. 区分协议（水平）与服务（垂直）

**易错点**：

1. 单位换算：网络传输用十进制，存储用二进制
2. 不同模型中相同功能的实现层次不同（如流量控制）
3. 延迟计算要注意选择合适的方法
4. 数据通信是网络最基础、最重要的功能

**计算相关**：电路交换延迟计算、分组交换延迟计算（两种方法）、单位换算
