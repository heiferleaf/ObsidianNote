索引，相当于数据库中数据的目录，作用是快速查找指定数据。加快的核心原理是减少IO时间，更快定位到数据

## 索引的分类

### 按照用户视角分类

| 索引类别 | 关键字            | 核心区别                   |
| ---- | -------------- | ---------------------- |
| 主键索引 | `primary key`  | 一张表只能定义一个主键索引，索引键值唯一非空 |
| 唯一索引 | `unique key`   | 一张表可以定义多个，索引键值唯一       |
| 普通索引 | `key / index`  | 一张表可以定义多个，索引键值不唯一      |
| 组合索引 | `INDEX(a,b,c)` | 多个字段组合为索引，遵循最左匹配原则     |
> [!NOTE] 最左匹配原则
> 组合索引 INDEX(a,b,c) 的物理数据结构
> - 按a进行排序
> - a相同的，按b进行排序
> - ab相同的，按c进行排序
> 这种结构决定了如果只使用b作为查询条件时，索引失效，因为b的数据列，不一定有序

### 按照物理存储方法分类

| 索引类别        | 核心区别                                              |
| ----------- | ------------------------------------------------- |
| 聚簇索引（一级索引）  | 叶子节点存：索引值+完整数据。一张表只能有一个聚簇索引，聚簇索引叶子节点顺序就是实际存放数据的顺序 |
| 非聚簇索引（二级索引） | 叶子节点存：索引值+记录ID，需要通过记录ID，回表在查询一次                   |
> [!NOTE] 回表查询
> 使用二级索引进行数据表的查询操作时，比如
> `SELECT * FROM user WHERE username = 'Gemini'`
> 1. 在`username`的索引文件（二级索引）中，通过B+ Tree查找到 Gemini 的叶子节点，拿到记录ID
> 2. 在主键的索引文件中，查找对应的记录ID
> 这里的第二步操作就是回表查询

### 按照索引查询时，内存中使用的数据结构分类

| 索引类别    | 核心点                                         |
| ------- | ------------------------------------------- |
| B+ 树索引  | 支持范围查询，叶子节点存放数据，并且叶子节点之间有指针串联               |
| Hash 索引 | 将索引键值通过哈希算法，得到定位。**支持精准匹配**，**不支持范围查询**     |
| 全文索引    | 通过倒排索引，支持对键进行分词，通过部分内容定位到记录ID，不需要像B+树，完整匹配键 |
> [!NOTE] 倒排索引
> 两部分组成：词典和倒排列表
> 1. **词典**：索引值中，不重复单词的集合
> 	通过对索引键值的文本进行分词，比如“我爱AI”，分为“我爱”、“AI”
> 	分词后的单词，通过B+ Tree或者HashMap进行组织，实现快速的查找分词
> 2. **倒排列表**：每一个分词后，都有一个列表，存放包含该分词的记录ID
>    
> 使用索引过程
> 1. 对查找的where条件的文本做分词
> 2. 在词典中查找分词
> 3. 将所有的分词的列表交集处理
> 4. 获得记录ID列表
> 5. 回表查询


但是不是所有情况都适合使用索引。

## 适合使用索引的情况

1. 数据列经常用于查询
2. 数据列中的值区分度较大，索引的利用效率高
	如果数据列中都是一个数值，完全没有在这个列上进行条件筛选、查找数据的必要
3. 用于排序和分组的数据列，因为索引文件是有序排序，所以排序和分组的效率比较高
4. 查找的列被索引覆盖
	命中索引文件之后，就可以直接返回，不需要再去查数据行
5. 多表查询的关联列，可以提高表连接的效率，按照归并排序的算法运行

## 不适合使用索引的场景

1. 数据行很少，全表扫描效率会比索引查询快
2. 区分度很低的数据列
3. 更新频繁的数据列，维护索引文件代码大
4. 不常用做查询的数据列


## 索引会失效的场景

1. 隐式的类型转换
	phone = 187... ， 数据库中是字符串类型，但是查询的时候，没有使用“”包围，会隐式类型转换，同时索引会失效
2. 模糊匹配
	name = "%john"，左边的%会让索引失效
3. 不等于的判断
	索引用作查询条件，使用的运算符都是 < > <= >=
	面对 < > != not 会失效
