## 特征

由结点构成，除了根节点，所有结点有父亲结点。除了叶子结点，所有的结点都有子节点。

## 术语

1. 度
	- 结点的度指结点的子节点数量
	- 树的度指树中结点度的最大值
2. 深度
	- 结点的深度是结点到叶子节点的最长路径长度 + 1
	- 树的深度是根节点的深度
3. 高度
	距离根节点的路径长度，根节点高度是0
4. 内部结点
	除去根结点和叶子结点之外的结点
5. 分支结点
	除去叶子结点之外的结点
6. 二叉树
	结点的子节点数量是2
7. 满二叉树
	二叉树中，高度为 `h` 的结点数量为 $2^h$， 深度为 `d` 表示结点数量为 $2^d - 1$
8. 完全二叉树（完美二叉树）
	二叉树中，深度为 `d` , 只有 `d-1` 和 `d-2` 高度的结点可能是叶子结点，并且 `d-2` 高度的结点必须先有左节点才会有右节点

## 特殊变形

### BST 二叉搜索树

- 特点：
	- 中序遍历有序
	- 查找满足二分查找的特征

- 插入核心代码
```Java
  find(k) {
	  if 当前查找结点的键 < k 在右子树中查找
	  if 当前相等 标记找到了，返回
	  if 当前查找结点的键 > k 在左子树中查找
  }
  put(K k, V v) {
	  根节点为空，直接设置根节点
	  
	  结点 = find(k);
	  如果找到标记 为 0，说明修改value
	  否则返回的结点是需要插入子节点的结点
	  根据标志对应插入左节点或者右节点
  }
```


- 删除结点核心代码
	思路是统一处理为结点的删除
	对于既有左子树又有右子树的情况，找到左子树的最右侧结点，将删除结点的键值替换，接下去也是删除这个最右侧结点的逻辑
	对于结点的删除，`C++` 还需要我们手动释放内存，但是 `Java` 的话，只要没有栈区的引用变量指向堆区内存，`GC` 会自动回收
	在删除结点的逻辑判断中，需要注意删除结点是否为根节点，根节点的删除还涉及到树的 `root` 结点的引用地址改变
```Java
public void remove(K key) {
	TreeNode<K, V> node = find(key, root);
	if(node == null) return;
	// 统一处理为删除结点至多有一个子树的情况
	if(node.left != null && node.right != null) {
		// 左子树的最右侧
		TreeNode<K, V> child = node.left;
		while(child.right != null) {
			child = child.right;
		}
		node.key = child.key;
		node.value = child.value;
		node = child;// 接下来需要删除node结点
	}
	// 判断是左子树存在还是右子树存在还是都不存在
	TreeNode<K, V> child = (node.left == null) ? node.right : node.left;
	TreeNode<K, V> p = node.parent;
	if(child == null) {
		// 无子树的情况
		if(p == null) // 根节点
			root = null;
		else if(p.left == node) {
			p.left = child;
		} else {
			p.right = child;
		}
		chlid.parent = p;
	} else if(p == null) {
		// 删除有子树的根节点
		root = child;
		child.parent = null;
	} else if(p.left == null) {
		// 有子树的非根节点
		p.left = child;
		child.parent = p;
	}else {
		p.right = child;
		child.parent = p;
	}
}
public TreeNode<K, V> get(K k, TreeNode<K, V> node) {
	if(node == null) return null;
	if(node.key.compartor(k) < 0 ) {
		return get(k, node.right);
	}
	else if(node.key.compartor(k) == 0) {
		return node;
	}else
		return get(k, node.left);
}
```

### AVT 平衡二叉树

> 二叉搜索树的形状和结点的插入顺序关系巨大。一种中序序列，对应很多种树形态
> 其中，高度越矮的树，在平均查找性能上最优，为 $\log_2n,n是结点数量$

平衡二叉树，通过保证节点平衡 （左结点深度和右节点深度差绝对值不超过1），保证树尽可能地矮。

#### 旋转方案

1. `标志1` ：失衡节点的平衡状态，`L` 为左侧重， `R` 为右侧重
2. `标志2` ：(`L`失衡节点的左结点 | `R`失衡节点的右节点) 的失衡状态， `L | R`

根据标志组合
1. `LL` ： 右旋
	![[树 2026-01-24 19.35.57.excalidraw]]
2. `RR` : 左旋
	![[树 2026-01-24 19.41.32.excalidraw]]
3. `LR` : 左旋失衡结点左节点 + 右旋失衡节点
4. `RL` ： 右旋失衡节点右节点 + 左旋失衡结点

```
在上述 BST 插入结点之后，从插入结点的父节点开始，向上查找
结点更新height，检测有无失衡，失衡按照对应的旋转方式调整
因为时刻保持平衡，所以旋转一次后，整个树一定是平衡的，可以直接退出循环
而在结点的删除中，子树的平衡不保证整个树的平衡，需要向上检查到根节点为止
```
### 红黑树

在 `AVT` 的矮 和 维护 `AVT` 矮 之间做一个权衡。
- 比`AVT`树，维护的平衡性稍弱，但是维护开销也相应弱一些
- 比普通的二叉树，平衡性强，查找效率高

#### 遵循规则

- **每个节点要么是红色，要么是黑色。**
    
- **根节点永远是黑色的。**
    
- **每个叶子节点（NIL 节点，空节点）是黑色的。**
    
- **红色节点不能相连**（即红色节点的子节点必须是黑色的）。
    
- **从任一节点到其每个叶子的所有简单路径，都包含相同数目的黑色节点**（这被称为“黑高平衡”）。

#### 插入算法逻辑

- **我是红，爸是红，叔叔也是红** ➔ 爸叔变黑，爷变红，向上走。
    
- **我是红，爸是红，叔叔是黑的** ➔
    
    - 如果是弯的（LR/RL）➔ 旋直它。
        
    - 如果是直的（LL/RR）➔ 爷旋一下，爸变黑，爷变红。
        
- **最后别忘了：根节点永远涂黑。**
