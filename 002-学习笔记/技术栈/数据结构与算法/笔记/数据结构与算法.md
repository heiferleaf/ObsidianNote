## 什么是数据结构

对于一个实际的问题，比如
$\textbf{背包问题：给定一系列的物品，物品有自己的重量W和价值V，求大小为M的背包最多存放多少价值的物品}$
想要求解这个问题，可以分为三个步骤：
1. 分析问题，确定数据模型
2. 选定算法
3. 编写程序

数据模型确定：
- 需要存储的内容：二维的状态`dp[i][j]`，含义为前i个物品在容量为j的背包中，最大装填价值
- 需要满足的约束：背包装填物品重量 `<=` 背包容量
- 求解目标：`dp[N][M]`
选定算法：
	使用动态规划算法，确定状态转移方程
编写程序

有些数据模型可以用数学表达式表示，但是更多的实际问题无法用数学表达式表示，这需要我们从数据入手分析。

**数据**：对事物的符号表述，具体形式可以是图片、视频、数字，在计算机中最常用的就是数字表示的数据，任何其他形式，也会转为二进制数据进行处理
**数据元素**：是数据的单位，比如人、电脑、手机、铅笔盒
**数据项**：是具有独立含义的数据最小单位，若干数据项组成数据元素，比如人由姓名，年龄，性别等等组成
**数据对象**：数据元素组成的集合
**数据结构**：数据对象的集合，包括数据元素之间的关系
	1. 数据的逻辑结构：以集合、线性关系、树、图抽象表示的数据结构，每一种逻辑结构中，数据元素之间有关系，比如线性的前驱后继，树的父子结点，图的出边入边
	2. 数据的存储结构：顺序存储，链式存储，索引存储，哈希存储
	3. 数据运算：在数据对象上的操作，比如插入、删除、查找数据对象中的数据元素

通过对数据进行分析，分析逻辑结构和对应的物理存储结构，设计数据的运算，包括针对逻辑结构的抽象运算，和针对物理结构的具体实现，这样得到的数据结构，也是一种数据模型。

## 什么是算法

我们通过对问题进行分析，或者对问题的数据进行分析，用数学表达式或者数据结构表示出数据模型。算法就是在数据模型的基础上，通过定义有限步的确定操作，将数据模型转为期望答案的这个过程。
算法因而具有5个特性：
1. 有穷性
2. 可行性：计算机硬件可以执行
3. 确定性
4. 输入性：零个或者多个输入
5. 输出型：一个或者多个输出

```Java
// 违反了有穷性
void exam1() {
	int n = 2;
	while(n % 2 == 0) {
		n += 2;
	}
}
// 违反了可行性
void exam2() {
	int x, y = 0;
	x = 5 / y; // 计算机对于除法的硬件操作，会检查被除数是否为零
}
```


## 算法的衡量

一个好的算法，满足
1. 正确性
2. 可使用性，容易被用户使用，比如调用条件不能太复杂
3. 可读性
4. 健壮性：对于边界数据，异常数据，能正确处理
5. 时间性能、空间性能。

### 性能的衡量

一个算法，可以由控制部分和数据操作部分描述。控制部分包括顺序、分支、循环。而算法的执行时间，就由控制结构和原操作综合决定（控制结构可以决定原操作的执行次数）。

- **问题规模n**：指算法输入的数据量的大小，比如排序10个数据，n就为10。
- **算法频度$T(n)$**：是问题规模$n$的函数，表示原操作的执行次数。
- 算法执行时间：严格来说，会$\geq 原操作执行时间 \times T(n)$；一般性，就用$T(n)$表示，忽略原操作的执行时间。

1. 对于$T(n)$的时间复杂程度表示
	考虑到$T(n)$的形式比较复杂，往往由多项组成，进一步用时间复杂度来衡量时间性能。
	时间复杂度是$T(n)$的数量级表示，考虑函数中增长速度最快的一项（当$默认规模n$无穷大时，$T(n)$的决定项），记为$T(n)=O(f(n))$
		1. 在没有循环结构，或者循环次数和问题规模无关的算法中，时间复杂度就是常数项$O(1)$
		2. 常见的时间复杂度大小关系$O(1) < O(\log_2 n) < O(\sqrt{n}) < O(n) < O(n \log_2 n) < O(n^2) < O(n^3) < O(2^n) < O(n!)$
		3. $O(f_1(n)) + O(f_2(n)) = \max{O(f_1(n)), O(f_2(n))}$ 
		4. $O(f_1(n)) \times O(f_2(n)) = O(f_1(n) \times f_2(n))$ 
		5. 考虑问题规模为$n$的固定，数据会有多种形式，比如数组中数据的顺序。所有可能的示例集合记为$D_n$，对于任意的$I \in D_n$，记$P(I)$为$I$出现的概率，$\sum\limits_{I \in D_n}P(I) = 1$
			1. 平均时间复杂度：$A(n) = \sum\limits_{i \in D_n}P(I) \times T(I)$
			2. 最好时间复杂度：$B(n)=\min_{I \in D_n}{T(I)}$
			3. 最坏时间复杂度：$W(n)=\max_{I \in D_n}{T(I)}$

2. 对于算法的空间复杂度分析
	空间存储情况，只需要考虑算法中的临时变量，不用考虑形参。这是因为形参会在调用处被算入存储空间
	空间复杂度也用问题规模$n$的函数表示
