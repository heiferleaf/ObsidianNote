## ArrayList 和 Array 有什么区别？ArrayList 和 LinkedList 的区别是什么？

1. ArrayList和Array的区别
	- 类型
		- ArrayList是集合框架中的一个类，实现了List接口，可以存放引用数据类型
		- Array是Java提供的一个数据结构，可以存放基本数据类型和引用数据类型
	- 扩容
		- ArrayList是动态的数组，支持在必要的时候，进行数据容量扩大和缩小
		- Array是静态的数组，数组大小在初始化指定后，不能再改变
	- 性能
		- ArrayList访问元素的性能较低
			- ==拿到ArrayList对象指向堆区中的地址==
			- ==拿到堆区中的存放的数组地址==
			- 计算索引对应数组元素的地址
			- 访问到元素
			- ==有时还要进行元素的拆箱和装箱==
		- Array访问元素性能较高，根据数组首地址+（索引 * 元素大小），得到元素地址
	- 方法
		- ArrayList提供了较多针对元素、数组的方法
		- Array较少
	- 泛型
		- Array可以支持泛型
		- ArrayList必须使用泛型
2. **ArrayList和LinkedList的区别**
	- 实现本质
		- ArrayList基于数组实现
		- LinkedList基于双向链表
	- 操作性能
		- ArrayList访问随机索引数据性能较好，删除/增加元素性能一般较低
		- LinkedList访问随机数据性能较差，删除/增加元素性能高
	- 内存占用
		- LinkedList需要额外的空间存放结点指针

## HashSet是如何保证元素不重复的

HashSet是集合框架中的一个类，实现了Set接口。特点是存放不重复的元素。

HashSet内部使用一个HashMap，元素不重复的原理就是HashMap的key不重复的原理。

通过hashCode方法，将key映射到一个数组索引，如果数组该索引存在元素，则通过equals方法，判断是否相等，如果相等就失败，不相等就可以通过单向链表或者红黑表组织索引重复项

## Java中的HashMap了解吗？HashMap 的底层实现是什么?

HashMap是存放键值对的数据结构，底层实现采用数组 + 单向链表 + 红黑树。

HashMap存放的键值对，会通过哈希函数，将key映射到数组索引，再通过equals方法，判断是否和已存在的元素相等。

当数组长度大于等于64，并且某一个索引位置的元素个数达到8，会将该处索引的元素，用红黑树进行组织。

当某处索引原本采用红黑树进行组织时，随着元素删除，个数低于6，会改回单向链表组织

## HashMap 的扩容机制是怎样的？

当HashMap存放元素变多，达到阈值时，会进行存放数组的扩容操作。
> 阈值一般为：`当前容量 * 装填因子（一般0.75）`

**扩容操作**
- 一般将数组大小设置为原来的两倍
- 将HashMap中存放的所有元素重新进行哈希，所以扩容前后的数组元素排序很大可能不一样

**扩容目的**
- 减少哈希冲突：装填元素越多，虽然利用度会提高，但是对于新元素的添加，很可能会哈希到已经存放元素的索引位置。
- 提高访问速度：当哈希冲突发生，通过单向链表和红黑树组织索引重复元素，会因为存放元素数量增多，导致访问速度降低。通过增加容量，减少哈希冲突，减少链表和树的存放元素，提高访问性能

## TreeMap 和 HashMap 的区别是什么？

- HashMap: 基于哈希表实现，通过将key哈希映射到数组索引，得到键值对的存放位置，数组中元素往往是无序的。
- TreeMap：基于红黑树实现，通过key的比较，在红黑树中找到创建结点的位置。集合中元素的Key往往是按照自然排序规则或者初始化时的比较规则排序的

在查找元素的性能上
- HashMap:O(1)
- TreeMap:O(logn)

## HashMap 和 HashTable 的区别？

1. 线程安全性
	1. HashMap:线程不安全，多线程中会有同步问题
	2. HashTable：线程安全，大多方法中有synchronized，但是还是推荐使用ConcurrentHashMap，细粒度锁，并发性高
2. null
	1. HashSet：允许有一个null键和多个null值
	2. HashTable：不允许null键和值
		如果有null键值，取元素得到null的时候，就不知道是不存在元素还是存在元素了，HashMap有containsKey方法
3. 性能
	1. `HashMap`：因为线程不安全，所以性能比较高
	2. `Hashtable`：由于方法同步，多线程开销较大，性能通常较低

> `HashTable` 是一个历史遗留类，现在已经不使用了

## HashMap 的 put 方法的流程是怎样的？

1. key 哈希 -> 索引
2. 数组索引处没有元素，添加到索引处
3. 数组索引处有元素，第一个元素equals判断，不相等
	1. 单向链表查找
	2. 红黑树查找
	3. 如果能找到，修改，找不到，添加，判断是否有单向链表改为红黑树
4. 判断数组长度是否要扩容
