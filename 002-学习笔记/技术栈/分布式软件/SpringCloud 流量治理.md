# Sentinel 原理部分

> Sentinel 是阿里的一个流量治理组件，关注于：当下游服务出问题或流量太大时，怎么保护整个系统不雪崩。实现分布式系统的**高可用**

## （1） 限流

当一个服务被请求太多次，超过了阈值，可以直接拒绝或者排队等待

限流保护的是自己

## （2） 熔断

当调用下游服务的失败率超过阈值，自动"断开"这条调用链路，不再往下游发请求，过一段时间再试探性恢复。
```
正常调用 -> 失败次数超过阈值 -> 直接 fallback 返回一个类似稍后再试，不把请求交给下游处理 -> 等一段时候后，发送一个请求试探一下是否可行了 -> 如果服务成功，就恢复正常使用
```

熔断保护的是下游服务

## （3） 降级

服务出问题时，返回一个兜底响应而不是报错。比如推荐服务挂了，就返回默认热门商品列表。

降级保护的是**用户体验**，让用户看到降级结果而不是一个错误页


## Sentinel

**核心是 资源 + 规则**
- 资源是服务中接口、或者代码、方法
- 规则是为资源定义的保护条件
> 规定的定义和对资源的访问是解耦的，资源所代表的业务逻辑，完全不需要关注规则是什么，当规则修改时，资源不需要任何修改

所需依赖

```xml
<!-- 流量控制 sentinel -->
<dependency>  
    <groupId>com.alibaba.cloud</groupId>  
    <artifactId>spring-cloud-starter-alibaba-sentinel</artifactId>  
</dependency>
```

所需配置

```yml
spring:
	cloud:
		sentinel:  
		  transport:  
		    dashboard: localhost:8858  
		    port: 8720
```

资源注册，在方法上注解 @SentinelResource
- value ： 资源命名
- fallback ： 熔断发生时，降级服务
- blockhandler ： 限流时，降级服务

流量控制：在 Dashboard 中，配置资源的流控规则
	限流条件( QPS 每秒请求数 ) | (线程并发数)
	限流类型：
		1. 直接：限制自己的访问流量
		2. 关联：当A流量激增时候，限制B的流量
		3. 链路：指定限制的入口资源，A->C,B->C,C的限流可以指定只限制从A进入的
	限流对象：方法 | 方法中的某一个参数值
		某一个商品是爆款，就在限制方法参数值 (@RequestParam Long productId) 通过让爆款商品能接收更大的流量。配置
```
参数索引：0          ← 第0个参数，即 productId
单机阈值：100        ← 默认所有值 QPS=100
例外项：
  参数值=1001，阈值=10   ← 爆款单独限更严
  参数值=1002，阈值=200  ← VIP商品放宽
```


熔断处理

配置服务的熔断规则
- 熔断配置位置：A->B,如果想配置B服务出现异常时，进行熔断，需要在A中配置
- 熔断配置触发条件：
	- 慢响应（响应时间超过指定时间，记为慢响应） | 异常比例 | 异常数量
	- 统计时间（统计一段时间内的异常情况）
	- 最小请求数（在统计时间内，需要达到最小请求数内，在进行阈值判断）
	- 比例阈值（当异常情况达到占比时，才认为熔断）
	- 熔断时间

通过 Feign 指定限流、熔断后服务降级
@FeignClient（ fallback = 服务降级函数的实现类.class | fallbackFactory = FallbackFactory\<FeignClient.class>）
