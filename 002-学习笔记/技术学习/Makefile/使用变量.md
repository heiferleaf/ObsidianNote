## 1. 变量是什么？

- **概念**：变量就是用来存储文本的“盒子”，你定义了一个名字，给它赋值，之后在Makefile中用这个名字替代文本。
    
- **区别于C/C++宏**：C/C++宏是预处理替换，展开后固定不变；Makefile变量是“惰性展开”或“立即展开”（后面会说），可以重新赋值，灵活多变。
    
- **变量作用域**：变量可以用在目标名、依赖列表、命令行甚至另一个变量的定义里。
    

### 变量名规则

- 只能包含字母（大小写区分）、数字、下划线（_）。
    
- 不允许出现特殊字符，如 `: # = 空格`等。
    
- 推荐命名方式：使用大小写混合（比如`MakeFlags`），减少与系统变量冲突。

## 2. 如何声明和使用变量？

### 声明变量

- 语法示例：
    
    ```makefile
    VAR = value
    ```
    
- `VAR` 是变量名，`value`是字符串值，可以是任意文本。
    

### 使用变量

- 使用时需要加美元符号 `$`，一般用括号括起来，保证识别：
    
    ```makefile
    $(VAR)
    ```
    
- 也可以写成`${VAR}`。
    

### 为什么要括号？

- 括号让Make准确识别变量边界，避免变量名和其他字符连在一起出错。
    

### 特殊符号

- 如果你要在命令或变量值中写`$`字符本身，要写成`$$`，因为`$`是Make变量标记。
    

### 举例说明

```makefile
objects = main.o foo.o utils.o
program: $(objects)
    cc -o program $(objects)
```

- `objects`变量保存了一个文件列表。
    
- `program`目标依赖`$(objects)`中的文件。
    
- 编译命令用变量值来替代，避免重复写文件名。

## 3. 变量的两种赋值方式

### 递归赋值（`=`）

- 赋值时**不立即展开右边内容**，而是在使用变量时才展开（惰性求值）。
    
- 这意味着右边可以引用其它变量，不管这些变量是在前面还是后面定义的。
    
- 优点：灵活，后定义的变量也能被引用。
    
- 缺点：如果不小心，可能导致无限递归（变量互相引用），导致Make报错。
    

**示例：**

```makefile
foo = $(bar)
bar = $(ugh)
ugh = Hello
all:
    echo $(foo)
```

- `$(foo)`引用`$(bar)`，`$(bar)`引用`$(ugh)`，`$(ugh)`是`Hello`。
    
- 使用时Make会递归展开，最终`$(foo)`变成`Hello`。
    

---

### 简单赋值（`:=`）

- 赋值时**立即展开右边内容**（即时求值）。
    
- 右边只能引用之前定义的变量，后定义的无效。
    
- 优点：避免递归展开，运行效率更高。
    
- 缺点：不支持后引用变量。
    

**示例：**

```makefile
x := foo
y := $(x) bar
x := later
```

- 这里，`y`会被立即展开成`foo bar`，而不是`later bar`。
    

---

### 对比总结

|赋值方式|何时展开右边表达式|是否允许后定义变量|性能|适用场景|
|---|---|---|---|---|
|`=` (递归)|使用时|允许|较慢，可能递归风险|变量依赖灵活，复杂定义|
|`:=` (简单)|赋值时|不允许|快，安全|确定值，不依赖后续变量|

## 4. 空格与空字符串变量定义

- Makefile中很难直接定义空格字符为变量值。
    
- 通常做法是定义一个空字符串变量，再通过赋值空字符串变量 + 注释写法定义空格变量。
    

**示例：**

```makefile
nullstring :=
space := $(nullstring) # end of line
```

- `nullstring`为空字符串。
    
- `space`通过“空字符串 + 注释”技巧得到一个空格值。
    
- 该技巧利用`#`注释截断变量定义行，避免尾部空格被误读。

## 5. 条件赋值（`?=`）

- 只有当变量之前未定义时，`?=`才会赋值。
    
- 可以用来提供变量默认值。
    

**等价写法：**

```makefile
ifeq ($(origin FOO), undefined)
    FOO = bar
endif
```

## 6. 变量值替换

### 简单替换

- 格式：
    
    ```makefile
    $(var:a=b)
    ```
    
- 含义：将变量`var`中所有以`a`结尾的单词替换为`b`结尾。
    

**示例：**

```makefile
foo := a.o b.o c.o
bar := $(foo:.o=.c)  # bar = a.c b.c c.c
```

### 模式替换

- 使用`%`作为通配符，匹配变量中对应模式部分。
    
- 格式：
    
    ```makefile
    $(var:%.o=%.c)
    ```
    
- 功能同上，但更灵活。

## 7. 多层变量引用（间接引用）

- 通过多层`$()`实现间接引用。
    
- 意思是变量的值也是另一个变量名，然后再取那个变量的值。
    

**示例：**

```makefile
x = y
y = z
a := $($(x))  # 先取$(x) = y，然后$(y) = z，最终a = z
```

### 更复杂示例

```makefile
x = variable1
variable2 := Hello
y = $(subst 1,2,$(x))
z = y
a := $($($(z)))  
```

- 解析：
    
    - `$(x)` = `variable1`
        
    - `subst 1,2,$(x)` 将`variable1`的`1`替换为`2`，得到`variable2`
        
    - `z = y`
        
    - `a := $($($(z))) = $($(y)) = $(variable2) = Hello`

## 8. 组合变量名

- 利用变量拼接生成新的变量名，然后取值。
    

**示例：**

```makefile
first_second = Hello
a = first
b = second
all = $($a_$b)  # 相当于 $(first_second)，值是 Hello
```

---

## 9. 追加变量值（`+=`）

- 用`+=`给变量追加新的文本内容。
    
- 如果变量之前未定义，`+=`相当于定义并赋值。
    
- 如果之前用`=`定义，追加操作是递归赋值；如果之前用`:=`定义，追加操作保持立即展开。
    

**示例：**

```makefile
objects = main.o foo.o bar.o
objects += another.o
```

- 结果`objects`值是`main.o foo.o bar.o another.o`。

## 10. override 指令

- 当变量被命令行或者环境变量覆盖时，Makefile中同名变量赋值无效。
    
- 使用`override`强制覆盖命令行和环境变量的值。
    

**示例：**

```makefile
override CFLAGS = -O2
override CFLAGS += -Wall
```

## 11. 多行变量定义（`define`）

- 用`define`和`endef`定义多行变量。
    
- 适合定义多行命令序列。
    
- 命令必须用Tab开头，才能被Make识别为命令执行。
    

**示例：**

```makefile
define print_messages
echo "Hello"
echo $(NAME)
endef
```

## 12. 环境变量与Make变量

- Make启动时，会导入系统环境变量为Make变量。
    
- 如果Makefile中定义了同名变量，会覆盖环境变量。
    
- 使用`-e`参数时，环境变量优先。
    
- 子Makefile继承父Makefile的变量，但只有命令行变量默认传递，普通变量需要`export`声明才能传递。
    

---

## 13. 目标变量（Target-specific Variables）

- 定义只在某个目标及其依赖规则中有效的变量。
    
- 可与全局变量同名，互不干扰。
    

**语法示例：**

```makefile
prog : CFLAGS = -g

prog : prog.o foo.o bar.o
    $(CC) $(CFLAGS) prog.o foo.o bar.o

prog.o : prog.c
    $(CC) $(CFLAGS) prog.c
```

- 在`prog`目标及其依赖中，`CFLAGS`都是`-g`，其他地方不受影响。
    

---

## 14. 模式变量（Pattern-specific Variables）

- 类似目标变量，但作用于符合特定模式的所有目标。
    
- 常用于定义某类文件的特定变量。
    

**示例：**

```makefile
%.o : CFLAGS = -O2
```

- 所有`.o`文件在编译时使用`-O2`优化选项。
